# ShopXO 微信商城小程序底座搭建 Spec

## Why
在本地工作区搭建一套基于 ShopXO 官方开源项目的可运行微信商城小程序底座，实现后端可运行、数据库可初始化、管理后台可访问、uni-app 可编译到微信小程序，为后续业务二开提供基础环境。

## What Changes
- 创建 `/shopxo-backend`、`/shopxo-uniapp`、`/docs` 三大目录
- 从 GitHub 拉取 ShopXO 后端仓库（gongfuxiang/shopxo）到 `/shopxo-backend`
- 从 GitHub 拉取 ShopXO uni-app 仓库（gongfuxiang/shopxo-uniapp）到 `/shopxo-uniapp`
- 安装后端 Composer 依赖
- 配置后端数据库连接（MySQL）
- 通过安装向导或手动方式初始化数据库
- 配置 Web 服务器（PHP 内置开发服务器或 Nginx）使后端可访问
- 安装 uni-app 前端 npm 依赖
- 配置 uni-app 前端 API 地址指向本地后端
- 确保前端可编译运行到微信小程序/H5
- 生成项目文档

## Impact
- Affected code: 新建整个项目结构，不修改已有代码
- Affected systems: 本地 PHP 运行环境、MySQL 数据库、Node.js 环境
- 依赖外部服务: MySQL >= 5.7、PHP >= 8.0、Node.js >= 14、Composer、Git

## ADDED Requirements

### Requirement: 目录结构创建
系统 SHALL 在工作区根目录下创建以下目录结构：
- `/shopxo-backend` — 后端代码
- `/shopxo-uniapp` — uni-app 手机端代码
- `/docs` — 项目文档

#### Scenario: 目录创建成功
- **WHEN** 执行目录创建命令
- **THEN** 三个目录均存在且可写入

### Requirement: 官方仓库代码拉取
系统 SHALL 从 GitHub 拉取 ShopXO 官方仓库代码，优先使用稳定分支（master/main）。

#### Scenario: 后端仓库拉取成功
- **WHEN** 执行 `git clone https://github.com/gongfuxiang/shopxo.git shopxo-backend`
- **THEN** `/shopxo-backend` 目录包含完整的 ShopXO 后端代码，包含 `app/`、`config/`、`public/`、`vendor/` 等目录

#### Scenario: uni-app 仓库拉取成功
- **WHEN** 执行 `git clone https://github.com/gongfuxiang/shopxo-uniapp.git shopxo-uniapp`
- **THEN** `/shopxo-uniapp` 目录包含完整的 uni-app 项目代码，包含 `pages/`、`common/`、`static/` 等目录

### Requirement: 后端依赖安装与运行
系统 SHALL 安装后端 Composer 依赖并确保后端可通过 PHP 内置服务器或配置的 Web 服务器运行。

#### Scenario: Composer 依赖安装成功
- **WHEN** 在 `/shopxo-backend` 目录执行 `composer install`
- **THEN** `vendor/` 目录生成且无报错

#### Scenario: 后端可启动
- **WHEN** 启动 PHP 内置开发服务器指向 `public/` 目录
- **THEN** 浏览器访问 `http://localhost:8080` 可返回页面（安装向导或首页）

### Requirement: 数据库初始化
系统 SHALL 确保 MySQL 数据库可被 ShopXO 后端正确连接和初始化。

#### Scenario: 数据库配置正确
- **WHEN** 后端配置文件中填写了正确的 MySQL 连接信息
- **THEN** 后端可成功连接数据库

#### Scenario: 安装向导可执行
- **WHEN** 浏览器访问 `http://localhost:8080/install`
- **THEN** 显示 ShopXO 安装向导页面，可按步骤完成数据库初始化

### Requirement: uni-app 前端配置与运行
系统 SHALL 配置 uni-app 前端 API 地址指向本地后端，并确保前端可编译运行。

#### Scenario: API 地址配置成功
- **WHEN** 在 uni-app 配置文件中设置 `request_url` 为本地后端 API 地址
- **THEN** 前端请求可正确到达后端

#### Scenario: 前端可编译运行
- **WHEN** 执行 `npm run dev:h5` 或编译微信小程序
- **THEN** 前端可正常启动并显示页面

### Requirement: 项目文档生成
系统 SHALL 生成以下文档：
1. `/docs/01-项目结构说明.md` — 目录结构与各模块说明
2. `/docs/02-本地启动说明.md` — 完整的本地启动步骤
3. `/docs/03-已完成事项与阻塞项.md` — 已完成的工作和待解决的阻塞项

#### Scenario: 文档内容完整
- **WHEN** 文档生成完成
- **THEN** 每份文档包含完整的、可操作的信息，包括启动命令、访问地址、默认账号等

## 假设与风险

### 假设
1. 本地已安装 Git、PHP >= 8.0（含必要扩展）、MySQL >= 5.7、Composer、Node.js >= 14
2. MySQL 默认运行在 127.0.0.1:3306，root 用户可用
3. 使用 PHP 内置开发服务器进行本地开发调试，不依赖 Nginx/Apache
4. 微信小程序开发使用微信开发者工具，本地开发阶段关闭域名校验
5. 后端运行端口为 8080，前端 H5 运行端口为 5173（默认）

### 风险
1. **PHP 扩展缺失**：ShopXO 需要 fileinfo、gd、curl 等扩展，Windows 环境可能默认未启用
2. **Composer 依赖安装失败**：网络问题可能导致 Composer 包下载失败，需配置国内镜像
3. **npm 依赖安装失败**：同理需配置 npm 国内镜像
4. **跨域问题**：H5 开发模式下前后端不同端口，需配置代理或 CORS
5. **版本匹配**：uni-app 端版本需与后端版本匹配，需确认两者使用同一版本的代码
6. **安装向导权限**：安装向导需要写入配置文件的权限，Windows 下通常无问题
