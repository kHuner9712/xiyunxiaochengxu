# 备份与恢复说明

## 一、需要备份的内容

### 1.1 项目代码

| 目录 | 说明 | 备份方式 |
|------|------|----------|
| shopxo-backend/app/ | 后端自定义代码 | 整目录复制 |
| shopxo-backend/config/ | 配置文件 | 整目录复制 |
| shopxo-backend/public/ | 入口文件 | 整目录复制 |
| shopxo-uniapp/ | 小程序前端代码 | 整目录复制 |
| docker/ | Docker 配置 | 整目录复制 |
| docker-compose.yml | Docker 编排 | 单文件复制 |
| docs/ | 项目文档+SQL | 整目录复制 |

### 1.2 数据库

| 数据库 | 说明 | 包含的关键表 |
|--------|------|-------------|
| shopxo | 全部业务数据 | sxo_activity, sxo_activity_signup, sxo_invite_reward, sxo_user, sxo_admin, sxo_power, sxo_config, sxo_goods 等 |

### 1.3 运行时数据

| 目录 | 说明 | 是否必须备份 |
|------|------|-------------|
| shopxo-backend/runtime/ | 缓存/日志 | 否（可重建） |
| shopxo-backend/public/static/upload/ | 上传文件 | 是（如有用户上传） |

## 二、数据库导出命令

### 2.1 全库导出

```powershell
docker exec shopxo-mysql mysqldump -uroot -proot123456 shopxo > shopxo_backup_20260417.sql
```

### 2.2 仅导出自定义表

```powershell
docker exec shopxo-mysql mysqldump -uroot -proot123456 shopxo sxo_activity sxo_activity_signup sxo_invite_reward > muying_tables_backup_20260417.sql
```

### 2.3 仅导出配置数据

```powershell
docker exec shopxo-mysql mysqldump -uroot -proot123456 shopxo sxo_config sxo_power > config_backup_20260417.sql
```

### 2.4 导出演示数据（适合新环境初始化）

已有现成文件：`docs/muying-demo-data.sql`

## 三、项目恢复步骤

### 3.1 全新环境恢复

**前置条件**：已安装 Docker Desktop

```powershell
# 1. 恢复代码
cd c:\Users\15864\Desktop\yunxixiaochengxu
# 确保以下目录存在：shopxo-backend/, shopxo-uniapp/, docker/, docs/, docker-compose.yml

# 2. 启动 Docker
docker compose up -d

# 3. 等待 MySQL 就绪（约30秒）
docker exec shopxo-mysql mysql -uroot -proot123456 -e "SELECT 1;"

# 4. 导入数据库（如果是空库）
docker exec -i shopxo-mysql mysql -uroot -proot123456 shopxo < shopxo_backup_20260417.sql

# 5. 清除缓存
docker exec shopxo-php sh -c "rm -rf /var/www/html/runtime/cache/shopxo/*"

# 6. 验证
curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/
# 应返回 200
```

### 3.2 仅恢复自定义表

```powershell
# 适用于已有 ShopXO 基础数据的环境
# 唯一迁移入口：muying-final-migration.sql（旧脚本已废弃）
docker exec -i shopxo-mysql mysql -uroot -proot123456 shopxo < docs/muying-final-migration.sql
docker exec -i shopxo-mysql mysql -uroot -proot123456 shopxo < docs/muying-demo-data.sql
```

### 3.3 恢复后必做检查

1. 访问 http://localhost:8080/ 确认首页正常
2. 访问 http://localhost:8080/adminwlmqhs.php 确认后台可登录（admin/admin123）
3. 检查「运营」菜单下是否有：活动管理、报名管理、邀请管理、数据报表
4. 执行测试 API：`curl http://localhost:8080/api.php?s=/muyingtest/signuplist.html`

## 四、定期备份建议

| 频率 | 备份内容 | 命令 |
|------|----------|------|
| 每日 | 数据库 | `docker exec shopxo-mysql mysqldump -uroot -proot123456 shopxo > backup_$(date +%Y%m%d).sql` |
| 每周 | 代码+数据库 | 压缩整个项目目录 |
| 上线前 | 全量备份 | 代码+数据库+上传文件 |

## 五、紧急回滚

如果更新后出现问题：

1. 停止容器：`docker compose down`
2. 恢复代码：从备份目录复制
3. 恢复数据库：`docker exec -i shopxo-mysql mysql -uroot -proot123456 shopxo < backup.sql`
4. 重启：`docker compose up -d`
5. 清缓存：`docker exec shopxo-php sh -c "rm -rf /var/www/html/runtime/cache/shopxo/*"`
