## 二、微信开发者工具配置

### 2.1 打开项目

1. 打开微信开发者工具
2. 导入项目，选择 `shopxo-uniapp` 目录
3. AppID 可使用测试号或填写真实 AppID

### 2.2 关闭域名校验（开发阶段必须）

1. 微信开发者工具右上角点击「详情」
2. 选择「本地设置」标签
3. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」

**这是开发阶段必须的操作**，因为本地开发使用 HTTP + 非标准域名。

### 2.3 确认 request_url

App.vue 中已配置：
```javascript
request_url: 'http://192.168.1.217:8080/'
```

如果 IP 变化，修改此值后重新编译。

## 三、验证步骤

### 3.1 登录

1. 小程序启动后，进入「我的」页面
2. 点击登录按钮
3. 输入测试账号：testuser01 / 123456

**验证 API**：
```
POST http://192.168.1.217:8080/api.php?s=/user/login.html
参数: type=username&accounts=testuser01&pwd=123456
```

**已验证结果**：code=0，登录成功

## 四、IP 变化处理

如果本机 IP 变化（如切换网络），需要：

1. 查看新 IP：
```powershell
ipconfig | findstr "IPv4"
```
找到局域网 IP（通常是 192.168.x.x）

2. 修改 `shopxo-uniapp/App.vue` 中的 request_url：
```javascript
request_url: 'http://新IP:8080/'
```

3. 重新编译小程序

## 五、上线时需要修改的配置

1. request_url 改为正式域名（HTTPS）
2. 在微信公众平台配置 request 合法域名
3. 删除 Muyingtest 控制器
4. 关闭开发者工具的「不校验合法域名」选项

## 六、常见问题

### Q: 小程序请求失败，提示 "request:fail"
A: 检查：
1. 后端服务是否运行（访问 http://192.168.1.217:8080/ 确认）
2. 是否勾选了「不校验合法域名」
3. IP 是否正确（ipconfig 确认）

### Q: 登录时提示验证码错误
A: 已关闭图形验证码，如果仍出现，检查配置：
```sql
SELECT value FROM sxo_config WHERE name='home_user_login_img_verify_state';
-- 应该为 0
```

### Q: 页面空白
A: 检查控制台错误信息，确认 request_url 配置正确
