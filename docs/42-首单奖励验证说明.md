# 首单奖励验证说明

## 一、验证原理

首单奖励流程：
1. 老用户（inviter_id）生成邀请码
2. 新用户（invitee_id）通过邀请码注册 → 触发注册奖励
3. 新用户完成首单 → 触发首单奖励
4. 邀请人获得积分

## 二、无需真实支付的测试方式

通过 Muyingtest 控制器的 FullFlow API，可以模拟完整的邀请→注册→首单流程，无需真实支付。

### 2.1 一键全流程测试

```
GET http://localhost:8080/api.php?s=/muyingtest/fullflow.html&inviter_id=1&invitee_id=4
```

参数说明：
- inviter_id: 邀请人用户ID（默认1）
- invitee_id: 被邀请人用户ID（默认4）

### 2.2 已验证证据（2026-04-17 真实测试）

### 2.3 分步测试

## 三、奖励配置

奖励值在 sxo_config 表中配置：

| 配置项 | 说明 | 当前值 |
|--------|------|--------|
| muying_invite_register_reward | 注册奖励积分 | 100 |
| muying_invite_first_order_reward | 首单奖励积分 | 200 |

修改方式：
```sql
UPDATE sxo_config SET value='200' WHERE name='muying_invite_register_reward';
UPDATE sxo_config SET value='500' WHERE name='muying_invite_first_order_reward';
```

## 四、已修复的 Bug

### Bug: IntegralService::IntegralLogAdd() 不存在

**问题**：InviteService::GrantReward() 调用了不存在的 `IntegralService::IntegralLogAdd()` 方法

**原因**：ShopXO 的 IntegralService 中正确方法名为 `UserIntegralLogAdd`，参数签名也不同

**修复**：
- 文件：`app/service/InviteService.php`
- 修改：将 `IntegralLogAdd($integral_data)` 改为 `UserIntegralLogAdd($user_id, $original_integral, $operation_integral, $msg, $type, $operation_id)`
- 修复时间：2026-04-17

## 五、生产环境接入说明

真实场景中，首单奖励应在订单支付成功后触发：

1. 找到订单支付成功的回调位置
2. 在支付成功后调用 `InviteService::OnFirstOrder(['user_id' => $order_user_id])`
3. 确保只触发一次（OnFirstOrder 内部有幂等检查）

## 六、注意事项

1. Muyingtest 控制器仅用于测试，上线前必须删除
2. FullFlow API 会删除被邀请人的旧邀请记录再重新创建，不要对生产数据使用
3. 邀请码算法 `md5($user_id . 'muying_invite')` 无盐值，生产环境应改用更安全的方式
