# 禧孕母婴用品私域商城小程序 - 信息架构

## 1. 文档概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 禧孕母婴用品私域商城小程序 |
| 运营主体 | 禧孕文化传媒有限公司 |
| 平台性质 | 甲方自营商城（非多商户平台） |
| 文档版本 | v1.0 |
| 文档目的 | 定义小程序端及管理后台的完整信息架构，明确页面层级、数据流向、导航设计与信息展示优先级，为产品设计与技术开发提供统一的结构参考 |
| 适用范围 | 小程序端（C 端）、管理后台（B 端） |

## 2. 小程序信息架构

### 2.1 底部 Tab 导航

小程序底部固定 5 个 Tab 入口，覆盖用户核心购物路径：

| 序号 | Tab 名称 | 图标风格 | 核心定位 | 页面路径 |
|------|----------|----------|----------|----------|
| 1 | 首页 | 首页图标 | 发现与导购，聚合运营位与推荐内容 | `/pages/home/index` |
| 2 | 分类 | 分类图标 | 按品类快速查找商品 | `/pages/category/index` |
| 3 | 活动 | 活动图标 | 营销活动集中展示与参与 | `/pages/activity/index` |
| 4 | 购物车 | 购物车图标（含数量角标） | 购物决策与结算入口 | `/pages/cart/index` |
| 5 | 我的 | 用户图标 | 个人中心与账户管理 | `/pages/mine/index` |

**Tab 导航规则：**
- 底部 Tab 在所有一级页面常驻显示
- 进入二级及更深页面时，底部 Tab 隐藏，顶部显示返回导航
- 购物车 Tab 角标实时显示已加入商品数量，数量为 0 时不显示角标
- 未登录用户可浏览首页、分类、活动页；点击购物车、我的页时引导登录

### 2.2 首页模块架构

首页自上而下由以下模块组成，按运营优先级排列：

| 模块序号 | 模块名称 | 模块说明 | 是否需登录 |
|----------|----------|----------|------------|
| 1 | 搜索栏 | 顶部固定搜索入口，支持关键词搜索、热门搜索词推荐、搜索历史 | 否 |
| 2 | Banner 轮播 | 运营配置的轮播图，可跳转商品详情、活动页、内容页等 | 否 |
| 3 | 功能入口区 | 图标导航矩阵，包含：新人礼包、限时折扣、积分中心、会员中心、宝宝档案、领券中心等（运营可配置） | 部分需登录 |
| 4 | 按宝宝月龄推荐 | 根据用户宝宝档案中的月龄智能推荐适配商品；无宝宝档案时显示引导添加入口 | 是 |
| 5 | 热门推荐 | 运营配置的热销商品列表，横向滑动或网格展示 | 否 |
| 6 | 新品上架 | 按上架时间倒序展示新品，支持"查看更多"跳转 | 否 |
| 7 | 活动专区入口 | 当前进行中的重点活动卡片，点击进入活动详情 | 否 |
| 8 | 猜你喜欢 | 基于用户浏览/购买行为的个性化推荐瀑布流，无限滚动加载 | 否（登录后更精准） |

**首页模块详细说明：**

#### 2.2.1 搜索栏
- 位置：页面顶部固定，不随页面滚动
- 交互：点击进入搜索页，展示搜索框、热门搜索词、搜索历史
- 搜索结果页支持按销量、价格、上架时间排序
- 搜索结果支持按分类、品牌筛选

#### 2.2.2 Banner 轮播
- 数量：3-8 张，运营可配置
- 自动轮播间隔：3 秒
- 支持手动左右滑动
- 每张 Banner 可配置跳转链接（商品详情/活动页/分类页/外部链接）

#### 2.2.3 功能入口区
- 布局：一行 4-5 个图标，最多两行
- 图标下方显示文字标签
- 运营可在后台配置显示哪些入口及排序
- 默认入口：新人礼包、限时折扣、积分中心、会员中心、宝宝档案、领券中心

#### 2.2.4 按宝宝月龄推荐
- 数据来源：用户宝宝档案中的出生日期或预产期
- 推荐逻辑：根据宝宝当前月龄匹配对应月龄段的适用商品
- 月龄分段：0-3 月、4-6 月、7-12 月、1-2 岁、2-3 岁、3 岁以上
- 未添加宝宝档案时显示"添加宝宝，获取专属推荐"引导卡片
- 未登录时显示登录引导

#### 2.2.5 热门推荐
- 展示形式：横向滑动卡片或 2 列网格
- 商品信息：商品图片、名称、价格、销量标签
- 数据来源：运营后台配置 + 销量自动排序
- 点击进入商品详情页

#### 2.2.6 新品上架
- 展示形式：2 列网格
- 商品信息：商品图片、名称、价格、"新品"标签
- 右上角"查看更多"跳转新品列表页
- 按上架时间倒序排列

#### 2.2.7 活动专区入口
- 展示形式：大图卡片，1-2 个活动
- 显示活动名称、活动时间、核心优惠信息
- 点击进入对应活动详情页

#### 2.2.8 猜你喜欢
- 展示形式：瀑布流双列布局
- 商品信息：商品图片、名称、价格、标签（热销/新品/折扣）
- 推荐算法：协同过滤 + 内容推荐 + 热度加权
- 无限滚动加载，每次加载 10-20 个商品
- 未登录时按热度推荐

### 2.3 分类页架构

分类页采用左右分栏布局：

| 区域 | 内容 | 交互说明 |
|------|------|----------|
| 左侧 | 一级分类列表 | 纵向排列，当前选中项高亮，点击切换右侧内容 |
| 右侧 | 二三级分类 + 商品 | 顶部展示二级分类网格，下方展示三级分类及推荐商品 |

**分类页详细结构：**

```
┌──────────────┬────────────────────────────┐
│              │  二级分类网格（图标+名称）    │
│  一级分类1    │  ┌────┐ ┌────┐ ┌────┐     │
│  ─────────   │  │二级1│ │二级2│ │二级3│     │
│  一级分类2    │  └────┘ └────┘ └────┘     │
│  一级分类3    │  ┌────┐ ┌────┐ ┌────┐     │
│  一级分类4    │  │二级4│ │二级5│ │二级6│     │
│  一级分类5    │  └────┘ └────┘ └────┘     │
│  ...         │                            │
│              │  三级分类列表                │
│              │  ├─ 三级分类1               │
│              │  ├─ 三级分类2               │
│              │  └─ 三级分类3               │
│              │                            │
│              │  推荐商品（2列网格）          │
│              │  ┌────────┐ ┌────────┐     │
│              │  │ 商品1   │ │ 商品2   │     │
│              │  └────────┘ └────────┘     │
└──────────────┴────────────────────────────┘
```

**分类层级设计：**
- 一级分类：如奶粉辅食、纸尿裤、洗护用品、童装童鞋、玩具早教、妈妈用品等
- 二级分类：如奶粉辅食下分为奶粉、辅食、营养品、零食等
- 三级分类：如奶粉下分为1段、2段、3段、4段等
- 分类图标：每个一级和二级分类均配有图标，后台可配置
- 点击二级/三级分类进入对应商品列表页

### 2.4 活动页架构

活动页集中展示所有进行中和即将开始的活动：

#### 2.4.1 活动分类 Tab

| Tab 名称 | 说明 |
|----------|------|
| 全部 | 展示所有类型的活动 |
| 限时折扣 | 限时特价商品活动 |
| 满减 | 满额减免活动（如满199减20） |
| 满赠 | 满额赠品活动 |
| 组合套餐 | 多件组合优惠套餐 |

#### 2.4.2 活动卡片列表

每个活动卡片包含以下信息：
- 活动封面图
- 活动名称
- 活动时间（开始时间 - 结束时间）
- 活动标签（限时/满减/满赠/套餐）
- 核心优惠信息摘要
- "立即参与"按钮

#### 2.4.3 倒计时组件

- 限时折扣活动显示倒计时：天/时/分/秒
- 未开始活动显示"即将开始"及距离开始的时间
- 已结束活动显示"已结束"，置灰处理
- 倒计时每秒刷新

### 2.5 购物车页架构

购物车页面自上而下分为三个区域：

#### 2.5.1 购物车商品列表

| 元素 | 说明 |
|------|------|
| 店铺名称 | "禧孕自营"标识（自营商城统一标识） |
| 商品图片 | 商品缩略图，点击跳转商品详情 |
| 商品名称 | 最多显示两行，超出省略 |
| 商品规格 | 已选规格信息（如：1段/900g） |
| 商品价格 | 单价，如有活动价则显示原价划线 + 活动价 |
| 数量控制 | 加减按钮，支持手动输入数量 |
| 选择框 | 单选/全选，用于结算选择 |
| 删除 | 左滑删除或长按删除 |
| 库存不足提示 | 库存不足时显示"库存不足"并置灰 |
| 失效商品 | 已下架/已失效商品单独分组展示，支持一键清理 |

#### 2.5.2 推荐商品

- 位置：购物车商品列表下方
- 展示形式：横向滑动商品卡片
- 推荐逻辑：基于购物车商品关联推荐
- 信息：商品图片、名称、价格、"加入购物车"按钮

#### 2.5.3 结算栏

- 位置：页面底部固定
- 内容：全选框 | 合计金额 | "去结算"按钮（显示已选商品数量）
- 优惠信息：已选商品可用的满减/优惠券提示
- 编辑模式：支持批量删除、移入收藏

## 3. 管理后台信息架构

### 3.1 后台主菜单

管理后台采用左侧边栏菜单导航，主菜单项如下：

| 序号 | 菜单名称 | 图标 | 核心定位 |
|------|----------|------|----------|
| 1 | 工作台 | 仪表盘 | 数据概览与快捷操作入口 |
| 2 | 商品中心 | 商品 | 商品全生命周期管理 |
| 3 | 订单中心 | 订单 | 订单与售后全流程管理 |
| 4 | 用户中心 | 用户 | 用户信息与会员管理 |
| 5 | 会员营销 | 营销 | 优惠券、活动、推荐位等营销工具 |
| 6 | 活动运营 | 活动 | 活动创建与效果分析 |
| 7 | 内容管理 | 内容 | 内容发布与素材管理 |
| 8 | 供应商管理 | 供应商 | 供应商信息管理 |
| 9 | 数据统计 | 图表 | 多维度数据分析 |
| 10 | 系统设置 | 设置 | 权限、配置与日志管理 |

---

### 3.2 每个主菜单的子菜单和页面

#### 3.2.1 工作台

| 子菜单 | 页面 | 页面说明 |
|--------|------|----------|
| 数据看板 | 工作台首页 | 展示今日/昨日/本月核心数据：GMV、订单数、客单价、新增用户数、热销商品 TOP10、待处理事项（待发货订单、待处理售后） |

**数据看板核心指标：**
- 实时数据：今日 GMV、今日订单数、今日访客数、今日新增用户
- 对比数据：与昨日/上周同日对比增减
- 趋势图表：近 7 天/30 天 GMV 与订单趋势
- 热销商品 TOP10 排行
- 待处理事项快捷入口：待发货订单数、待处理售后数、库存预警数

---

#### 3.2.2 商品中心

| 子菜单 | 页面 | 页面说明 |
|--------|------|----------|
| 商品列表 | 商品列表页 | 分页展示所有商品，支持按分类/品牌/状态/关键词筛选；支持批量上下架、批量修改价格 |
| 新增/编辑商品 | 商品编辑页 | 商品基本信息（名称/分类/品牌/简介）、规格设置（多规格/单规格）、价格库存、商品图片/详情图、运费模板、上架状态 |
| 商品分类 | 分类管理页 | 一二三级分类的增删改查、排序、图标配置；支持树形拖拽排序 |
| 品牌管理 | 品牌列表页 | 品牌增删改查、品牌 Logo、品牌描述、关联分类 |
| 库存管理 | 库存列表页 | 各商品 SKU 库存查看、库存调整、库存预警设置、库存变更日志 |

## 4. 数据流向

### 4.1 用户购物流程数据流

```
用户浏览商品
      │
      ▼
  商品详情页 ──── 查看商品信息（名称/价格/规格/评价）
      │
      ├─→ 直接购买 ──→ 选择规格/数量 ──→ 跳转下单确认页
      │
      └─→ 加入购物车 ──→ 购物车页 ──→ 选择商品 ──→ 跳转下单确认页
                                              │
                                              ▼
                                        下单确认页
                                    ┌─────────┴─────────┐
                                    │  选择收货地址       │
                                    │  选择优惠券         │
                                    │  计算运费           │
                                    │  计算优惠金额       │
                                    │  确认订单信息       │
                                    └─────────┬─────────┘
                                              │
                                              ▼
                                         创建订单
                                    ┌─────────┴─────────┐
                                    │  锁定库存           │
                                    │  锁定优惠券         │
                                    │  生成订单号         │
                                    │  订单状态：待付款    │
                                    └─────────┬─────────┘
                                              │
                                              ▼
                                         发起支付
                                    ┌─────────┴─────────┐
                                    │  调用微信支付       │
                                    │  等待支付结果       │
                                    └─────────┬─────────┘
                                              │
                                    ┌─────────┴─────────┐
                                    │                   │
                                 支付成功            支付失败/超时
                                    │                   │
                                    ▼                   ▼
                               支付结果页          订单保留待付款
                                    │              （30分钟自动取消）
                                    ▼
                              订单状态：待发货
                                    │
                                    ▼
                              管理员发货（填写物流信息）
                                    │
                                    ▼
                              订单状态：待收货
                                    │
                                    ▼
                              用户确认收货（手动/自动）
                                    │
                                    ▼
                              订单状态：已完成
                                    │
                              ┌─────┴─────┐
                              │           │
                           评价商品    申请售后
                              │           │
                              ▼           ▼
                          评价数据     售后流程
                                       │
                              ┌────────┴────────┐
                              │                 │
                          退款/退货退款       换货
                              │                 │
                              ▼                 ▼
                          退款到账          重新发货
```

**关键数据节点：**
- 浏览 → 商品浏览记录写入（用于推荐算法）
- 加购 → 购物车数据写入
- 下单 → 订单数据写入、库存锁定、优惠券锁定
- 支付 → 支付流水写入、订单状态更新、库存扣减、优惠券核销
- 发货 → 物流信息写入、订单状态更新
- 收货 → 订单状态更新、积分发放、成长值增加
- 售后 → 售后单写入、库存回补（退货时）、退款流水写入

### 4.2 会员成长数据流

```
用户注册/登录
      │
      ▼
  创建用户记录 ──── 初始等级：注册会员
      │               初始成长值：0
      │               初始积分：0
      │
      ▼
  用户行为产生成长值
  ┌───────────────────────────────────┐
  │  行为类型        │  成长值规则      │
  ├──────────────────┼────────────────┤
  │  消费1元         │  +1 成长值       │
  │  每日签到        │  +5 成长值       │
  │  评价商品        │  +10 成长值      │
  │  分享商品        │  +5 成长值       │
  │  完善宝宝档案    │  +50 成长值      │
  └───────────────────────────────────┘
      │
      ▼
  成长值累计 ──→ 判断是否达到升级阈值
      │
      ├─→ 未达到 ──→ 保持当前等级
      │
      └─→ 达到 ──→ 等级升级
                      │
                      ▼
                 更新会员等级
                      │
                      ▼
                 解锁对应权益
              ┌────────┴────────┐
              │  折扣权益        │
              │  专属客服        │
              │  优先发货        │
              │  生日礼包        │
              │  新品试用        │
              │  月度礼包        │
              └─────────────────┘
                      │
                      ▼
                 用户享受权益 ──→ 下单时自动应用折扣
```

**成长值规则说明：**
- 成长值只增不减（消费退款不扣除成长值）
- 等级升级实时生效
- 等级降级：自然月内消费未达保级标准，次月降一级
- 成长值明细记录每笔成长值变动

### 4.3 营销活动数据流

```
管理员创建活动
      │
      ▼
  活动基础信息写入 ──── 活动类型/名称/时间
      │
      ▼
  关联商品与规则
  ┌───────────────────────────────────────────┐
  │  限时折扣：选择商品 → 设置折扣价/折扣比例   │
  │  满减活动：设置满减阶梯（满199减20等）       │
  │  满赠活动：设置满额门槛 → 选择赠品           │
  │  组合套餐：选择组合商品 → 设置套餐价          │
  └───────────────────────────────────────────┘
      │
      ▼
  活动发布 ──→ 活动状态：进行中
      │
      ▼
  小程序端展示活动
      │
      ▼
  用户参与活动 ──→ 浏览活动商品
      │
      ▼
  用户下单
  ┌───────────────────────────────────────────┐
  │  优惠计算引擎                               │
  │  ├─ 检查商品是否参与活动                     │
  │  ├─ 计算活动优惠金额                        │
  │  ├─ 叠加优惠券（如规则允许）                  │
  │  ├─ 计算最终支付金额                        │
  │  └─ 记录优惠明细                            │
  └───────────────────────────────────────────┘
      │
      ▼
  创建订单 ──→ 订单关联活动信息
      │
      ▼
  活动数据统计 ──→ 更新活动参与人数/订单数/GMV
      │
      ▼
  活动结束 ──→ 活动状态：已结束
      │
      ▼
  活动效果分析 ──→ 生成活动报告
```

**优惠叠加规则：**
- 限时折扣与优惠券：可叠加（默认）
- 满减与优惠券：可叠加（默认）
- 同类型活动不叠加（如两个满减活动取最优）
- 组合套餐不与其他活动叠加
- 叠加规则可在后台活动配置中调整

```

      ▼

      ▼
  加载权限 ──→ 根据 Role 获取 Menu 权限列表

      ├─→ 无权限 ──→ 返回 403 拒绝访问
      │
      └─→ 有权限 ──→ 执行业务处理
                          │
                          ▼
                     业务逻辑处理
                  ┌────────┴────────┐
                  │  参数校验        │
                  │  业务规则校验    │
                  │  数据变更        │
                  │  事务提交        │
                  └────────┬────────┘
                           │
                           ▼
                     操作日志记录
                  ┌────────────────────┐
                  │  操作人             │
                  │  操作时间           │
                  │  操作模块           │
                  │  操作类型（增/删/改）│
                  │  操作前数据         │
                  │  操作后数据         │
                  │  IP 地址            │
                  └────────────────────┘
                           │
                           ▼
                     返回操作结果 ──→ 前端刷新页面
```

**操作日志记录范围：**
- 商品：新增/编辑/上下架/删除
- 订单：修改地址/取消订单/发货/备注
- 用户：修改信息/调整等级/调整积分
- 营销：创建/编辑/发布/结束活动、创建/发放优惠券
- 系统：管理员变更/角色变更/系统配置变更

## 5. 导航设计

### 5.1 小程序导航

#### 5.1.1 底部 Tab 导航

| 规则 | 说明 |
|------|------|
| 显示范围 | 5 个一级页面（首页/分类/活动/购物车/我的）常驻显示 |
| 隐藏规则 | 进入二级及更深页面时自动隐藏 |
| 角标规则 | 购物车 Tab 显示商品数量角标；我的 Tab 显示未读消息/待处理红点 |
| 切换动画 | 页面无刷新切换，保持各 Tab 页面状态 |

#### 5.1.2 页面内导航

| 导航类型 | 使用场景 | 说明 |
|----------|----------|------|
| 顶部返回栏 | 所有二级及以下页面 | 左侧返回按钮 + 页面标题，固定在页面顶部 |
| 返回逻辑 | 默认返回上一页 | 从 Tab 进入的页面，返回时回到对应 Tab 页 |
| 页面内锚点 | 商品详情页 | 顶部 Tab 切换（商品/评价/详情），点击滚动到对应区域 |
| 面包屑 | 不适用 | 小程序端不使用面包屑导航 |

#### 5.1.3 深层页面返回逻辑

| 场景 | 返回逻辑 |
|------|----------|
| 首页 → 商品详情 → 返回 | 返回首页 |
| 分类 → 商品列表 → 商品详情 → 返回 | 返回商品列表 |
| 购物车 → 下单确认 → 支付结果 → 返回 | 返回我的页（订单列表） |
| 活动 → 活动详情 → 商品详情 → 返回 | 返回活动详情 |
| 我的 → 订单列表 → 订单详情 → 返回 | 返回订单列表 |
| 搜索 → 搜索结果 → 商品详情 → 返回 | 返回搜索结果 |
| 支付结果页 | 不支持返回到下单确认页，提供"查看订单"和"返回首页"按钮 |

**特殊返回处理：**
- 页面栈最大深度 10 层，超过时最早入栈的页面被替换
- 支付结果页使用 `redirectTo` 跳转，不加入页面栈
- 分享进入的页面，返回时回到首页

### 5.2 后台导航

#### 5.2.1 侧边栏菜单

| 规则 | 说明 |
|------|------|
| 布局 | 左侧固定侧边栏，宽度 220px |
| 展示方式 | 一级菜单常驻展示，点击展开/收起二级菜单 |

#### 5.2.2 面包屑导航

| 规则 | 说明 |
|------|------|
| 位置 | 页面顶部，侧边栏右侧内容区上方 |
| 格式 | 首页 > 一级菜单 > 二级菜单 > 当前页面 |
| 交互 | 点击面包屑中的任意层级可跳转到对应页面 |

#### 5.2.3 标签页导航

|------|------|

| 标签数量 | 最多保留 10 个标签页，超出时自动关闭最早的 |
| 状态保持 | 切换标签页时保持各页面的筛选条件和滚动位置 |

## 6. 信息展示优先级

### 6.1 首页信息优先级

首页模块从上到下按运营优先级排列，优先级越高越靠上：

| 优先级 | 模块 | 排列逻辑 |
|--------|------|----------|
| P0 | 搜索栏 | 最高优先级，固定顶部，用户主动搜索需求最强 |
| P0 | Banner 轮播 | 运营核心推广位，承载活动/新品/品牌宣传 |
| P1 | 功能入口区 | 高频功能快捷入口，降低用户查找成本 |
| P1 | 按宝宝月龄推荐 | 差异化核心功能，体现母婴场景化推荐优势 |
| P2 | 热门推荐 | 销量驱动，引导用户购买热销商品 |
| P2 | 新品上架 | 新品曝光，满足用户尝新需求 |
| P2 | 活动专区入口 | 活动引流，提升活动参与率 |
| P3 | 猜你喜欢 | 长尾推荐，提升浏览深度和转化率 |

**优先级设计原则：**
- 用户主动需求（搜索）> 运营推广需求（Banner/功能入口）> 场景化推荐（月龄推荐）> 热度驱动（热门/新品）> 个性化推荐（猜你喜欢）
- 首页装修功能支持运营调整模块顺序，但搜索栏固定顶部不可移动
- 模块可配置显示/隐藏，但至少保留搜索栏 + Banner + 一个推荐模块

---

### 6.2 商品详情信息优先级

商品详情页从上到下按用户决策流程排列：

| 优先级 | 信息模块 | 排列逻辑 |
|--------|----------|----------|
| P0 | 商品主图轮播 | 视觉第一印象，用户最关注的商品外观 |
| P0 | 商品名称 | 核心识别信息 |
| P0 | 商品价格 | 购买决策最关键因素（原价 + 售价 + 活动价） |
| P0 | 活动标签 | 如有活动，显示活动信息与倒计时，制造紧迫感 |
| P1 | 规格选择 | 必须选择规格才能加购/购买 |
| P1 | 优惠信息 | 可用优惠券/满减信息，降低决策门槛 |
| P1 | 商品卖点标签 | 如"自营正品"/"7天无理由"/"顺丰发货"等信任标签 |
| P2 | 商品评价摘要 | 好评率 + 评价数量，社交证明 |
| P2 | 顶部 Tab 导航 | 商品/评价/详情三个 Tab 切换 |
| P2 | 商品评价列表 | 用户评价详情，辅助决策 |
| P3 | 商品图文详情 | 详细的商品介绍图文 |
| P3 | 商品参数 | 规格参数表 |
| P3 | 相关推荐 | 同类/关联商品推荐 |
| P0 | 底部操作栏 | 固定底部：客服 + 收藏 + 加入购物车 + 立即购买 |

**信息展示逻辑：**
- 首屏（不滚动）必须展示：主图 + 名称 + 价格 + 活动标签 + 优惠信息 + 信任标签
- 第二屏：评价摘要 + 规格选择入口
- 后续屏：评价列表 → 图文详情 → 参数 → 相关推荐
- 底部操作栏固定，始终可见

### 6.3 订单详情信息优先级

订单详情页从上到下按用户关注程度排列：

| 优先级 | 信息模块 | 排列逻辑 |
|--------|----------|----------|
| P0 | 订单状态 | 用户最关心的当前状态（待付款/待发货/待收货/已完成/售后中） |
| P0 | 状态操作提示 | 当前状态下的操作引导（如"请尽快付款"/"商家已发货"） |
| P0 | 物流信息 | 待收货状态下，用户最关心物流进度 |
| P1 | 收货地址 | 确认商品寄送地址是否正确 |
| P1 | 商品信息 | 订单中的商品列表（图片/名称/规格/价格/数量） |
| P1 | 金额明细 | 商品总额 + 运费 + 优惠金额 + 实付金额 |
| P2 | 订单信息 | 订单编号、下单时间、支付时间、发货时间 |
| P2 | 支付信息 | 支付方式、支付流水号 |
| P2 | 买家备注 | 下单时填写的备注信息 |
| P3 | 操作日志 | 订单状态变更的时间线记录 |

**不同订单状态的展示重点：**

| 订单状态 | 重点展示信息 | 操作按钮 |
|----------|-------------|----------|
| 待付款 | 倒计时提示、收货地址、商品信息、金额明细 | 去付款、取消订单 |
| 待发货 | 预计发货时间、商品信息 | 提醒发货、申请退款 |
| 待收货 | 物流进度、收货地址、商品信息 | 确认收货、查看物流、申请售后 |
| 已完成 | 商品信息、评价入口 | 评价、申请售后、再次购买 |
| 售后中 | 售后进度、售后原因 | 查看售后详情、撤销申请 |
| 已取消 | 取消原因、商品信息 | 再次购买 |
