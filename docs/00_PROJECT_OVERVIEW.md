# 禧孕母婴用品私域商城小程序 - 项目总览

## 1. 项目基本信息

| 项目 | 内容 |
|------|------|
| 项目名称 | 禧孕母婴用品私域商城小程序 |
| 项目性质 | 甲方自营商城（非多商户平台） |
| 运营主体 | 禧孕文化传媒有限公司 |
| 项目定位 | 母婴用品线上私域商城 |
| 目标用户 | 孕产妇、0-6岁婴幼儿家长 |

| 上线平台 | 微信小程序 |

## 2. 项目性质说明

本项目为**禧孕文化传媒有限公司自营商城**，绝非多商户平台。以下逐条说明：

### 2.1 不做商家入驻

- 平台不开放商家注册入口，不提供商家入驻申请流程
- 不存在"商家中心""商家管理"等面向第三方商家的功能模块
- 所有商品由甲方公司采购部统一选品、采购、定价后上架销售

### 2.2 不做厂家独立店铺

- 不为任何厂家/供应商开设独立店铺页面
- 用户端不展示"某某旗舰店""某某专营店"等店铺概念
- 所有商品统一在甲方商城品牌下展示，用户感知的是"禧孕"品牌，而非各供应商品牌

### 2.3 不做厂家前台经营

- 厂家/供应商无法登录小程序端进行任何操作
- 厂家/供应商无法自行上架、下架、修改商品信息
- 厂家/供应商无法查看前台用户评价、订单详情等敏感信息

### 2.4 不做平台撮合交易

- 本平台不是"淘宝""拼多多"式的撮合型交易平台
- 交易双方为：买方（用户）和卖方（禧孕文化传媒有限公司）
- 平台不收取佣金、技术服务费等平台费用
- 所有交易收入归甲方公司所有，甲方公司再与供应商按采购合同结算

### 2.5 统一管理范围

以下所有环节均由甲方公司统一管理：

- **商品管理**：选品、定价、上架、下架、库存调整
- **订单管理**：接单、发货、物流跟踪、订单状态流转
- **支付管理**：微信支付商户号由甲方公司持有，所有资金进入甲方对公账户
- **发货管理**：甲方公司统一安排发货，无论是自仓发货还是供应商代发
- **售后管理**：退换货、退款、投诉处理均由甲方客服团队统一处理
- **会员管理**：会员等级、积分、权益均由甲方统一制定规则
- **客服管理**：所有用户咨询由甲方客服团队统一接待
- **营销活动**：优惠券、限时折扣、满减等活动由甲方运营团队统一策划和执行

### 2.6 供应商角色定位

- 厂家/供应商仅作为**供货来源**，是甲方公司的上游合作伙伴
- 后台数据模型中记录 `supplier_id` 字段，其用途仅限于：
  - **内部管理**：标识某商品的供货来源，方便采购部门管理供应链
  - **采购结算**：根据供应商维度汇总采购金额，按合同约定周期结算货款
  - **货源追踪**：出现质量问题时，可追溯到具体供应商，便于追责和整改
- `supplier_id` 不会出现在用户端任何页面或接口返回中
- 供应商不拥有独立后台，不直接参与平台运营

## 3. 核心目标

### 3.1 沉淀私域用户

- 通过微信小程序天然社交属性，将公域流量（公众号、社群、朋友圈广告等）转化为私域用户
- 建立用户画像，实现精准触达和个性化推荐
- 通过会员体系增强用户粘性，降低获客成本

### 3.2 提高复购率

- 母婴用品属于高频刚需品类，天然具备复购属性
- 通过会员等级权益、积分体系、优惠券发放等手段激励复购
- 通过订阅提醒（如奶粉、纸尿裤定期购买提醒）提升复购频次

### 3.3 通过活动促进转化

- 定期策划限时折扣、满减、新人专享等营销活动
- 利用微信社群、公众号推文等私域渠道进行活动传播
- 通过活动数据复盘，持续优化活动策略

### 3.4 建立会员体系

- 设计多级会员等级（普通会员、银卡、金卡、黑金）
- 不同等级享受不同折扣、积分倍率、专属优惠等权益
- 会员等级根据消费金额自动升级，形成消费激励闭环

### 3.5 统一管理厂家供货商品

- 建立供应商管理模块，记录供应商基本信息、合作状态、结算方式
- 商品关联供应商，便于采购管理、库存追踪和货款结算
- 供应商评价体系，定期评估供应商服务质量

### 3.6 完成微信小程序线上交易闭环

- 用户在小程序内完成浏览→加购→下单→支付→收货→评价全流程
- 对接微信支付 V3，确保支付安全合规
- 建立完善的售后流程，保障用户权益

## 4. 系统组成

### 4.1 微信小程序用户端

- **技术栈**：uni-app + Vue3 + TypeScript
- **目标平台**：微信小程序
- **核心功能**：
  - 商品浏览与搜索
  - 购物车管理
  - 订单下单与支付
  - 订单跟踪与售后
  - 会员中心（等级、积分、优惠券）
  - 营销活动参与（限时折扣、满减、优惠券领取）
  - 收货地址管理
  - 客服咨询
- **设计原则**：
  - 界面简洁，操作流畅，适配母婴用户群体
  - 首页突出活动和推荐商品，提升转化
  - 支持微信授权登录，降低注册门槛

### 4.2 Web 管理后台

- **技术栈**：Vue3 + Vite + TypeScript + Element Plus
- **目标用户**：甲方运营团队、采购团队、客服团队、管理层
- **核心功能**：
  - 商品管理（SPU/SKU 管理、上下架、库存调整）
  - 订单管理（订单列表、发货、退款、售后处理）
  - 会员管理（会员列表、等级调整、积分操作）
  - 营销管理（优惠券、限时折扣、满减活动配置）
  - 供应商管理（供应商信息、合作状态、结算管理）
  - 内容管理（轮播图、公告、推荐位配置）
  - 数据看板（销售数据、用户数据、活动数据）
  - 系统设置（角色权限、操作日志、系统参数）
- **设计原则**：
  - 基于角色的权限控制（RBAC），不同岗位看到不同菜单
  - 操作留痕，关键操作记录日志
  - 数据看板实时更新，辅助运营决策

### 4.3 后端 API 服务

- **技术栈**：Node.js + NestJS + TypeScript
- **核心职责**：
  - 提供小程序端和管理后台的 RESTful API
  - 业务逻辑处理（订单、支付、库存、会员、活动等）
  - 微信支付 V3 对接
  - 微信小程序登录对接
  - 文件上传处理
  - 数据校验与错误处理
  - 接口鉴权与权限控制
- **设计原则**：
  - 模块化架构，按业务领域划分模块
  - 统一响应格式，统一错误码
  - 接口版本管理
  - 请求限流，防刷防爬

### 4.4 数据库

- **技术选型**：MySQL 8.0
- **核心数据**：
  - 用户数据（会员信息、地址、收藏）
  - 商品数据（SPU、SKU、分类、属性）
  - 订单数据（订单、支付、物流、售后）
  - 营销数据（优惠券、活动、积分记录）
  - 供应商数据（供应商信息、采购记录、结算记录）
  - 系统数据（管理员、角色、权限、操作日志）
- **设计原则**：
  - 遵循第三范式，适度反范式优化查询性能
  - 软删除策略，关键数据不物理删除
  - 完善的索引设计，保障查询性能

### 4.5 缓存

- **技术选型**：Redis 7
- **使用场景**：
  - 会话管理（用户登录态、管理员登录态）
  - 热点数据缓存（商品详情、分类列表、首页数据）
  - 库存预扣（限时折扣等高并发场景下的库存扣减）
  - 限流计数（接口访问频率限制）
  - 验证码存储（短信验证码、图形验证码）
  - 分布式锁（防止重复下单、库存超卖）

### 4.6 文件上传服务

- **当前方案**：本地文件上传
  - 文件存储在服务器本地磁盘
  - 通过 Nginx 静态资源服务提供访问
- **预留扩展**：
  - 预留腾讯云 COS 适配接口
  - 预留阿里云 OSS 适配接口
  - 采用策略模式设计，切换存储方式只需修改配置，无需改动业务代码
- **支持文件类型**：
  - 商品图片（JPG、PNG、WebP）
  - 商品详情图（JPG、PNG、WebP）
  - 轮播图（JPG、PNG、WebP）
  - 用户头像（JPG、PNG）
  - 售后凭证图片（JPG、PNG）

### 4.7 微信支付服务

- **技术选型**：微信支付 V3
- **核心功能**：
  - JSAPI 支付（小程序内支付）
  - 退款（全额退款、部分退款）
  - 支付回调通知处理
  - 退款回调通知处理
  - 对账（每日对账单下载与核对）
- **安全措施**：
  - 证书加密存储
  - 回调签名验证
  - 幂等性处理（防止重复支付、重复退款）
  - 敏感信息加密

### 4.8 运营数据看板

- **位置**：管理后台首页 / 独立数据页面
- **核心指标**：
  - 实时数据：今日订单数、今日销售额、今日新增用户数、在线用户数
  - 趋势数据：近7天/30天销售额趋势、订单量趋势、用户增长趋势
  - 商品数据：热销商品排行、库存预警
  - 用户数据：会员等级分布、复购率、客单价
  - 活动数据：活动参与人数、活动转化率、活动 ROI

## 5. 技术架构

### 5.1 技术栈总览

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| 小程序端 | uni-app + Vue3 + TypeScript | uni-app 跨端能力强，Vue3 组合式 API 更灵活，TypeScript 保障类型安全 |
| 管理后台 | Vue3 + Vite + TypeScript + Element Plus | Vue3 生态成熟，Vite 构建速度快，Element Plus 组件丰富适合后台系统 |
| 后端 API | Node.js + NestJS + TypeScript | 前后端语言统一（TypeScript），NestJS 模块化架构清晰，适合中大型项目 |
| 数据库 | MySQL 8.0 | 成熟稳定的关系型数据库，社区支持好，运维成本低 |
| 缓存 | Redis 7 | 高性能内存数据库，适合缓存、会话、限流等场景 |
| ORM | Prisma | 类型安全的 ORM，自动生成类型定义，迁移管理方便 |
| 文件存储 | 本地 + 预留云存储 | 初期降低成本，预留扩展能力 |
| 支付 | 微信支付 V3 | 微信小程序唯一支付方式，V3 接口更安全规范 |
| 登录 | 微信小程序登录 | 无需额外注册，降低用户使用门槛 |
| 部署 | Docker Compose + Nginx + HTTPS | 容器化部署便于运维，Nginx 反向代理和静态资源服务，HTTPS 保障安全 |
| 代码仓库 | 单仓 monorepo | 前后端代码统一管理，共享类型定义，便于版本协调 |

### 5.2 小程序端技术详情

- **框架**：uni-app（基于 Vue3）
- **语言**：TypeScript
- **状态管理**：Pinia
- **UI 组件库**：uni-ui / 自定义组件
- **网络请求**：基于 uni.request 封装，统一拦截器处理鉴权和错误
- **样式方案**：SCSS + rpx 适配
- **构建工具**：Vite（uni-app 内置）

### 5.3 管理后台技术详情

- **框架**：Vue3
- **构建工具**：Vite
- **语言**：TypeScript
- **UI 组件库**：Element Plus
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **网络请求**：Axios，统一拦截器处理鉴权和错误
- **图表**：ECharts
- **富文本**：WangEditor / TinyMCE（商品详情编辑）

### 5.4 后端 API 技术详情

- **运行时**：Node.js 18+ LTS
- **框架**：NestJS
- **语言**：TypeScript
- **ORM**：Prisma
- **验证**：class-validator + class-transformer
- **鉴权**：JWT（Access Token + Refresh Token）
- **API 文档**：Swagger / OpenAPI
- **日志**：Winston
- **任务调度**：@nestjs/schedule（定时任务，如订单超时自动取消）

### 5.5 部署架构

```
用户（微信小程序）
       │
       ▼

       │
       ├──► API 服务（NestJS，Docker 容器）

       │        ├──► MySQL（Docker 容器）
       │        ├──► Redis（Docker 容器）
```

## 7. 核心业务流程概览

### 7.1 用户购物流程

```
用户浏览首页/分类/搜索
       │
       ▼
查看商品详情（SPU → 选择规格 → SKU）
       │
       ▼
加入购物车 / 立即购买
       │
       ▼
确认订单（选择地址、优惠券、备注）
       │
       ▼
提交订单（库存预扣）
       │
       ▼
微信支付（JSAPI 支付）
       │
       ├── 支付成功 → 订单状态变更为"待发货"
       │                   │
       │                   ▼
       │              仓库发货 → 物流跟踪 → 用户确认收货
       │                   │
       │                   ▼
       │              订单完成 → 评价 → 积分发放
       │
       └── 支付失败/超时 → 订单取消 → 库存释放
```

### 7.2 会员成长流程

```
用户首次登录（微信授权）→ 自动注册为普通会员
       │
       ▼
消费累计 → 达到升级门槛 → 自动升级（银卡/金卡/黑金）
       │
       ▼
享受对应等级权益（折扣、积分倍率、专属优惠、生日礼等）
       │
       ▼
积分获取（消费、签到、活动）→ 积分兑换（优惠券、商品）
       │
       ▼
复购激励（会员专享价、积分抵扣、等级保护期等）
```

### 7.3 营销活动流程

```
运营团队创建活动（限时折扣/满减/优惠券）
       │
       ▼
活动审核上线
       │
       ▼
用户端展示活动入口（首页轮播、活动专区、弹窗等）
       │
       ▼
用户参与活动（领券/限时折扣下单/满足满减条件）
       │
       ▼
活动数据实时统计（参与人数、转化率、销售额）
       │
       ▼
活动结束 → 数据复盘 → 优化下一轮活动策略
```

### 7.4 供应商供货流程

```
采购团队新增供应商（录入信息、签订合同）
       │
       ▼
关联商品（设置 supplier_id、采购价、结算方式）
       │
       ▼
商品上架销售（用户下单）
       │
       ▼
订单发货（自仓发货 or 供应商代发）
       │
       ▼
采购结算（按周期汇总供应商订单，生成结算单）
       │
       ▼
财务对账 → 付款 → 结算完成
```

## 8. 项目约束与边界

### 8.1 不做的功能（明确排除）

| 约束项 | 说明 |
|--------|------|
| 不做多商户入驻 | 平台不开放第三方商家入驻，所有商品由甲方自营 |
| 不做厂家独立后台 | 供应商没有独立的管理后台，无法自行管理商品和订单 |
| 不做平台撮合 | 不是撮合型交易平台，交易双方为用户与甲方公司 |
| 不做第三方支付 | 仅支持微信支付，不支持支付宝、银行卡等第三方支付方式 |
| 不做 APP 端 | 仅开发微信小程序端和 Web 管理后台，不做 iOS/Android APP |
| 不做直播功能 | 不集成直播带货功能，商品展示以图文为主 |
| 不做社交功能 | 不做用户社区、动态发布、关注等社交功能，仅支持分享给好友/群 |
| 不做跨境商品 | 不涉及跨境电商、海关报关、海外仓等业务 |
| 不做虚拟商品 | 不做虚拟商品（充值卡、话费、会员卡等），仅做实物商品 |

### 8.2 技术约束

- 小程序包体积限制：主包不超过 2MB，总包不超过 20MB
- 微信支付要求：必须使用已认证的微信支付商户号
- 数据合规：遵守《个人信息保护法》《数据安全法》等法律法规
- 微信平台规范：遵守微信小程序平台运营规范和审核要求

### 8.3 业务约束

- 供应商信息不对外暴露，用户端不展示供应商名称
- 所有商品定价权归甲方公司，供应商无权干预
- 售后责任主体为甲方公司，不转嫁给供应商
- 会员数据归甲方公司所有，不与第三方共享

## 9. 术语定义

| 术语 | 英文 | 定义 |
|------|------|------|
| 私域 | Private Domain | 品牌或企业自主拥有的、可反复触达的用户流量池，如微信社群、公众号粉丝、小程序用户等。与公域流量（如搜索引擎、平台推荐）相对 |
| SKU | Stock Keeping Unit | 最小库存单元，指商品的具体规格组合。如"某品牌奶粉 3段 800g"是一个 SKU，代表可售卖、可库存管理的最小单位 |
| SPU | Standard Product Unit | 标准产品单元，指一类商品的抽象。如"某品牌奶粉"是一个 SPU，其下包含多个 SKU（1段/2段/3段，800g/1200g 等） |
| 供应商 | Supplier | 为甲方公司提供商品的厂家或经销商，是供货来源，不是平台商家。在系统中通过 supplier_id 标识，仅用于内部管理 |
| 会员等级 | Member Level | 根据用户消费金额或其他条件划分的等级体系，不同等级享受不同权益。如普通会员、银卡会员、金卡会员、黑金会员 |
| 积分 | Points | 用户通过消费、签到、参与活动等行为获得的虚拟奖励，可用于兑换优惠券或商品，是会员激励体系的重要组成部分 |
| 优惠券 | Coupon | 由运营团队发放的优惠凭证，用户在下单时可使用以抵扣部分金额。类型包括满减券、折扣券、无门槛券等 |
| 活动 | Campaign/Activity | 运营团队策划的营销活动，如限时折扣、满减满赠、新人专享、组合套餐等，旨在提升用户转化和复购 |
| 售后 | After-sale Service | 用户收货后发起的退换货、退款等服务请求。包括仅退款、退货退款、换货等类型 |
| 自营 | Self-operated | 由甲方公司自主采购、定价、销售和售后的经营模式，与平台撮合模式相对 |
| 代发 | Drop Shipping | 供应商根据甲方订单直接发货给用户的模式，甲方仍为交易主体，供应商仅负责物流配送 |
| 采购价 | Purchase Price | 甲方公司向供应商采购商品的价格，即进货成本价，不对用户可见 |
| 销售价 | Selling Price | 商品面向用户的销售价格，由甲方公司根据市场策略定价 |
| 结算 | Settlement | 甲方公司与供应商之间按合同约定周期进行货款结算的过程 |
| RBAC | Role-Based Access Control | 基于角色的访问控制，通过角色分配权限，用户通过关联角色获得对应权限 |
| SPU 属性 | SPU Attribute | 商品的规格维度，如"段位""容量"等，用于区分同一 SPU 下的不同 SKU |
| 软删除 | Soft Delete | 不物理删除数据库记录，而是通过标记字段（如 deleted_at）标识记录已删除，保留数据可追溯性 |
| 幂等性 | Idempotency | 同一操作执行多次与执行一次效果相同，在支付和订单场景中尤为重要，防止重复扣款或重复下单 |
| 库存预扣 | Stock Pre-deduction | 用户下单时先锁定库存，支付成功后正式扣减，支付超时或取消后释放锁定库存，防止超卖 |
| 组合式 API | Composition API | Vue3 提供的 API 风格，通过 setup 函数和组合函数组织逻辑，相比选项式 API 更灵活、更易于逻辑复用 |
