# 后台菜单与权限调整

## 一、新增菜单结构

### 1.1 菜单层级

```
运营（一级菜单，pid=0）
├── 活动管理（二级菜单）
│   ├── 活动详情（三级权限，is_show=0）
│   ├── 活动添加/编辑（三级权限，is_show=0）
│   ├── 活动保存（三级权限，is_show=0）
│   ├── 活动删除（三级权限，is_show=0）
│   └── 活动状态更新（三级权限，is_show=0）
├── 报名管理（二级菜单）
│   ├── 报名详情（三级权限，is_show=0）
│   ├── 签到核销（三级权限，is_show=0）
│   └── 导出数据（三级权限，is_show=0）
├── 邀请管理（二级菜单）
│   └── 邀请详情（三级权限，is_show=0）
└── 数据报表（二级菜单，无三级权限）
```

### 1.2 菜单参数说明

| 层级 | name | control | action | icon | sort | is_show |
|------|------|---------|--------|------|------|---------|
| 一级 | 运营 | activity | index | icon-admin-operation | 15 | 1 |
| 二级 | 活动管理 | activity | index | - | 1 | 1 |
| 三级 | 活动详情 | activity | detail | - | 1 | 0 |
| 三级 | 活动添加/编辑 | activity | saveinfo | - | 2 | 0 |
| 三级 | 活动保存 | activity | save | - | 3 | 0 |
| 三级 | 活动删除 | activity | delete | - | 4 | 0 |
| 三级 | 活动状态更新 | activity | statusupdate | - | 5 | 0 |
| 二级 | 报名管理 | activitysignup | index | - | 2 | 1 |
| 三级 | 报名详情 | activitysignup | detail | - | 1 | 0 |
| 三级 | 签到核销 | activitysignup | checkin | - | 2 | 0 |
| 三级 | 导出数据 | activitysignup | export | - | 3 | 0 |
| 二级 | 邀请管理 | invite | index | - | 3 | 1 |
| 三级 | 邀请详情 | invite | detail | - | 1 | 0 |
| 二级 | 数据报表 | muyingstat | index | - | 4 | 1 |

**说明**：
- `is_show=1`：在后台左侧菜单中可见
- `is_show=0`：不在菜单中显示，但作为权限控制点存在，用于按钮级权限判断
- `sort`：同级菜单排序，数值越大越靠前

## 三、隐藏菜单列表

一期MVP不需要以下功能菜单，通过 `is_show=0` 隐藏（不删除，保留后续启用能力）：

| 菜单名称 | control | 隐藏原因 |
|----------|---------|----------|
| 多商户 | shop | 一期单商户运营，无需多商户 |
| 商家入驻 | shop | 一期无商家入驻需求 |
| 分销管理 | distribution | 一期不做分销体系 |
| 直播管理 | weixinliveplayer | 一期不做直播 |
| 积分商城 | coin | 一期不做积分商城兑换 |

---

## 四、隐藏SQL说明

```sql
-- -----------------------------------------------------------
-- 4. 隐藏一期不需要的功能菜单
-- -----------------------------------------------------------
UPDATE `sxo_power` SET `is_show`=0 WHERE `name` IN ('多商户', '商家入驻', '分销管理', '直播管理', '积分商城') OR `control` IN ('shop', 'distribution', 'weixinliveplayer', 'coin');
```

**说明**：
- 使用 `is_show=0` 而非 DELETE，保留数据完整性，后续可随时恢复
- 同时按 `name` 和 `control` 匹配，确保覆盖所有相关菜单项
- 隐藏后菜单不在后台左侧导航显示，但权限记录仍存在
- 如果某个角色已分配了这些权限，权限仍然生效，只是入口不可见

**回滚方式**：

```sql
UPDATE `sxo_power` SET `is_show`=1 WHERE `name` IN ('多商户', '商家入驻', '分销管理', '直播管理', '积分商城') OR `control` IN ('shop', 'distribution', 'weixinliveplayer', 'coin');
```

## 五、权限缓存清除方法

ShopXO 使用缓存存储权限数据，修改 `sxo_power` 表后需要清除缓存才能生效。

### 5.1 后台界面清除

1. 登录管理后台：`http://localhost:8080/adminwlmqhs.php`
2. 进入：系统管理 → 缓存管理
3. 点击"清除所有缓存"

### 5.2 命令行清除

```powershell
# 删除运行时缓存目录
docker exec shopxo-php rm -rf /var/www/html/runtime/cache/*
docker exec shopxo-php rm -rf /var/www/html/runtime/temp/*
```

### 5.3 数据库方式清除

```sql
-- 清空ShopXO缓存表（如果使用了数据库缓存驱动）
TRUNCATE TABLE sxo_cache;
```

### 5.4 注意事项

- 修改菜单/权限后**必须**清除缓存，否则后台看不到新菜单
- 如果清缓存后仍不显示，检查管理员角色是否已分配新权限（见第六节）
- 浏览器也可能缓存后台页面，按 `Ctrl+F5` 强制刷新

## 六、管理员角色权限分配说明

### 6.1 权限分配流程

1. 登录管理后台：`http://localhost:8080/adminwlmqhs.php`
2. 进入：系统管理 → 角色管理
3. 找到"管理员"角色（或创建新角色），点击编辑
4. 在权限树中勾选"运营"及其所有子权限：
   - ✅ 运营
     - ✅ 活动管理（含所有三级权限）
     - ✅ 报名管理（含所有三级权限）
     - ✅ 邀请管理（含所有三级权限）
     - ✅ 数据报表
5. 保存

### 6.2 数据库直接分配（可选）

如果后台界面操作不便，可直接操作 `sxo_role_power` 关联表：

```sql
-- 查询管理员角色ID（通常id=1）
SELECT id, name FROM sxo_role WHERE name='管理员';

-- 查询新增的运营菜单权限ID
SELECT id, name, pid FROM sxo_power WHERE control IN ('activity', 'activitysignup', 'invite', 'muyingstat');

-- 为角色分配权限（假设角色ID=1，权限ID需根据实际查询结果填写）
-- 示例：INSERT INTO sxo_role_power (role_id, power_id) VALUES (1, {power_id});
```

**注意**：
- 直接操作数据库后必须清除缓存
- 确保一级菜单"运营"和所有子菜单/权限都被分配
- 如果只分配了二级菜单没分配三级权限，按钮级权限校验会失败

### 6.3 权限校验机制

ShopXO 后台权限校验通过 `AdminIsPower(control, action)` 函数实现：

- 在动态表格中，通过 `AdminIsPower('activity', 'delete')` 判断是否显示删除按钮
- 在控制器中，Base 控制器自动校验当前管理员是否有对应 control+action 的权限
- 未分配权限时，访问对应页面会返回"无权限"提示
