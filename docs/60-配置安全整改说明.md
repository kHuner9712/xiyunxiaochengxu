# 60-配置安全整改说明

## 改了哪些文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `.env.example` | 新建 | 根目录环境变量模板，含 Docker 和 uni-app 所需变量 |
| `docker-compose.yml` | 修改 | MySQL 密码/端口改为从 .env 读取，默认不暴露 3306 |
| `.gitignore` | 修改 | 新增 `.env` 忽略规则 |
| `shopxo-uniapp/common/js/config/dev.js` | 新建 | 前端开发环境配置，集中管理 request_url 和 static_url |
| `shopxo-uniapp/App.vue` | 修改 | request_url/static_url 改为从 dev.js 导入 |
| `shopxo-backend/.env.example` | 修改 | PASSWORD 改为占位符 |

## 为什么改

1. **docker-compose.yml** 中 MySQL root 密码硬编码为 `root123456`，数据库端口 3306 直接暴露到宿主机
2. **App.vue** 中 `request_url` 硬编码为 `http://192.168.1.217:8080/`，局域网 IP 不应出现在代码中
3. **shopxo-backend/.env.example** 中 PASSWORD 写的是真实密码 `root123456`
4. 以上问题导致：仓库公开后敏感信息泄露、新人无法按模板配置、不同开发者的 IP/密码无法灵活切换

## 迁移步骤

### 1. 创建 .env 文件
```bash
cp .env.example .env
```

### 2. 填写实际配置
编辑 `.env` 文件，将占位符替换为实际值：
```
MYSQL_ROOT_PASSWORD=你的MySQL密码
MYSQL_HOST_PORT=3306          # 如需暴露到宿主机，填写端口号；留空则不暴露
NGINX_HOST_PORT=8080
UNI_APP_REQUEST_URL=http://你的开发机IP:8080/
```

### 3. 修改前端开发配置
编辑 `shopxo-uniapp/common/js/config/dev.js`：
```javascript
export default {
    request_url: 'http://你的开发机IP:8080/',
    static_url: '',
};
```

### 4. 修改后端数据库配置
编辑 `shopxo-backend/.env`（从 .env.example 复制）：
```
PASSWORD = 你的MySQL密码
```

### 5. 重启 Docker
```bash
docker-compose down
docker-compose up -d
```

## 风险点

1. **已部署环境**：如果已有运行中的 Docker 环境，修改密码后需要重建 MySQL 容器或手动修改密码
2. **前端配置文件**：`dev.js` 不在 `.gitignore` 中，因为它只包含开发环境默认值（localhost），不含敏感信息。如果团队需要不同配置，每人本地修改即可
3. **MySQL 端口**：默认不暴露 3306 到宿主机，只允许容器间通信。如需用 Navicat 等工具连接，需在 `.env` 中设置 `MYSQL_HOST_PORT=3306`

## 如何验证

1. `grep -r "192.168.1.217" .` — 确认仓库中不再包含硬编码局域网 IP
2. `grep -r "root123456" .` — 确认仓库中不再包含硬编码密码
3. `cat .env.example` — 确认模板中只有占位符
4. `docker-compose config` — 确认 Docker Compose 能正确读取 .env 变量
5. 新人 clone 后按 `.env.example` 操作能正常启动
