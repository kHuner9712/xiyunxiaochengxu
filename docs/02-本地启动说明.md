# 本地启动说明

## 前置条件

- Docker Desktop 已安装并运行
- Git 已安装
- Node.js >= 14 已安装
- HBuilderX 已安装（用于 uni-app 开发）

## 一、后端启动

### 1. 启动 Docker 容器

```powershell
cd c:\Users\15864\Desktop\yunxixiaochengxu
docker compose up -d
```

首次启动会自动构建 PHP 镜像并拉取 Nginx/MySQL 镜像，约需 5-10 分钟。

### 2. 验证容器状态

```powershell
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

应看到三个容器均为 `Up` 状态：
- shopxo-nginx: 0.0.0.0:8080->80/tcp
- shopxo-php: 9000/tcp
- shopxo-mysql: 0.0.0.0:3306->3306/tcp

### 3. 访问后端

| 页面 | 地址 |
|------|------|
| 前台首页 | http://localhost:8080/ |
| 管理后台 | http://localhost:8080/adminwlmqhs.php |
| API 接口 | http://localhost:8080/api.php |

### 4. 默认账号

| 项目 | 值 |
|------|------|
| 管理员账号 | admin |
| 管理员密码 | admin123456 |
| MySQL 用户名 | root |
| MySQL 密码 | root123456 |
| MySQL 数据库 | shopxo |
| MySQL 端口 | 3306 |

### 5. 停止后端

```powershell
docker compose down
```

### 6. 重新构建（修改 Dockerfile 后）

```powershell
docker compose up -d --build
```

## 二、前端启动

### 方式一：HBuilderX（推荐）

1. 打开 HBuilderX
2. 文件 → 打开目录 → 选择 `shopxo-uniapp` 目录
3. 运行 → 运行到小程序模拟器 → 微信开发者工具
4. 或 运行 → 运行到浏览器 → Chrome（H5 模式）

### 方式二：命令行（需安装 @dcloudio/vue-cli）

```powershell
cd c:\Users\15864\Desktop\yunxixiaochengxu\shopxo-uniapp
npm install
# H5 模式
npm run dev:h5
# 微信小程序模式
npm run dev:mp-weixin
```

> 注意：当前项目为 HBuilderX 项目，package.json 中未配置 dev/build 脚本。
> 如需命令行运行，需先安装 @dcloudio/vue-cli 并迁移项目结构。

### 前端 API 配置

API 地址配置在 `shopxo-uniapp/App.vue` 第 10-13 行：

```javascript
request_url: 'http://localhost:8080/',
static_url: 'http://localhost:8080/',
```

如需切换后端地址，修改此两行即可。

## 三、微信小程序开发

### 1. 准备工作

- 安装微信开发者工具
- 在微信公众平台注册小程序，获取 AppID
- 修改 `shopxo-uniapp/manifest.json` 中 `mp-weixin.appid` 为你的 AppID

### 2. 当前配置

- AppID: `wxda7779770f53e901`（官方演示 AppID，需替换为你自己的）
- urlCheck: `false`（本地开发关闭域名校验）

### 3. 运行步骤

1. 在 HBuilderX 中打开 `shopxo-uniapp` 项目
2. 运行 → 运行到小程序模拟器 → 微信开发者工具
3. 微信开发者工具中关闭「不校验合法域名」选项（开发阶段）

### 4. 注意事项

- 微信小程序正式环境要求后端必须为 HTTPS
- 本地开发时在微信开发者工具中关闭域名校验即可
- `localhost` 地址只能在开发工具中使用，无法在真机预览

## 四、常见问题

### Q: Docker 容器启动失败？

```powershell
# 查看容器日志
docker logs shopxo-nginx
docker logs shopxo-php
docker logs shopxo-mysql

# 重启所有容器
docker compose restart
```

### Q: 后端页面 500 错误？

```powershell
# 查看 PHP 错误日志
docker exec shopxo-php cat /var/www/html/runtime/log/最新日期.log
```

### Q: 数据库连接失败？

```powershell
# 检查 MySQL 是否就绪
docker exec shopxo-mysql mysql -uroot -proot123456 -e "SELECT 1;"

# 检查数据库配置
docker exec shopxo-php cat /var/www/html/config/database.php
```

### Q: 如何重置安装？

```powershell
# 删除数据库配置文件
docker exec shopxo-php rm /var/www/html/config/database.php
docker exec shopxo-php rm /var/www/html/app/install/config/database.php

# 清空数据库
docker exec shopxo-mysql mysql -uroot -proot123456 -e "DROP DATABASE shopxo; CREATE DATABASE shopxo CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;"

# 重新访问安装向导
# http://localhost:8080/install.php
```

### Q: 如何修改 PHP 配置？

编辑 `docker/php/Dockerfile` 中的 `shopxo.ini` 部分，然后重新构建：

```powershell
docker compose up -d --build php
```
