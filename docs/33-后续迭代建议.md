# 后续迭代建议

> 按优先级排列，每个建议包含功能描述、影响模块、预估工作量、依赖关系

---

## P0 — 必须完成（上线前）

### P0-01: 配置生产环境域名和 HTTPS

| 项目 | 内容 |
|------|------|
| 功能描述 | 为生产环境配置正式域名、SSL 证书、Nginx HTTPS，确保微信小程序可正常请求后端 API |
| 影响模块 | docker/nginx/default.conf、docker-compose.yml、shopxo-uniapp/App.vue、微信公众平台服务器域名配置 |
| 预估工作量 | 0.5 天 |
| 依赖关系 | 域名已购买并备案、SSL 证书已申请 |
| 具体操作 | 1. 修改 Nginx 配置添加 443 监听和 SSL 证书路径；2. docker-compose.yml 添加 443 端口映射和证书卷挂载；3. App.vue 中 request_url 改为 https://域名/；4. 微信公众平台配置 request 合法域名；5. 测试 HTTPS 连通性 |
| 验收标准 | 小程序真机可正常请求 API，无域名校验报错 |

### P0-02: 微信小程序审核发布

| 项目 | 内容 |
|------|------|
| 功能描述 | 完成微信小程序审核发布流程，使小程序正式上线 |
| 影响模块 | shopxo-uniapp/manifest.json（AppID）、微信公众平台 |
| 预估工作量 | 1-3 天（含审核等待时间） |
| 依赖关系 | P0-01（HTTPS 配置）、小程序 AppID 已替换、小程序类目已选择 |
| 具体操作 | 1. 替换 manifest.json 中 AppID 为正式值；2. HBuilderX 发行小程序；3. 微信开发者工具上传代码；4. 填写版本号和更新说明；5. 提交审核；6. 审核通过后发布 |
| 验收标准 | 小程序正式上线，用户可通过微信搜索使用 |

### P0-03: 数据库密码改用环境变量

| 项目 | 内容 |
|------|------|
| 功能描述 | 将 `config/database.php` 中硬编码的数据库密码改为从环境变量读取，消除密码泄露风险 |
| 影响模块 | shopxo-backend/config/database.php、shopxo-backend/.env.example、.gitignore |
| 预估工作量 | 0.5 天 |
| 依赖关系 | 无 |
| 具体操作 | 1. database.php 中 `password` 改为 `env('DATABASE.PASSWORD', 'root123456')`；2. 确认 .env 文件中 DATABASE.PASSWORD 配置；3. 将 config/database.php 加入 .gitignore（或移除密码硬编码）；4. 生产环境 .env 中设置强密码 |
| 验收标准 | 数据库密码不再出现在代码仓库中，环境变量可正常覆盖 |

### P0-04: 邀请码性能优化

| 项目 | 内容 |
|------|------|
| 功能描述 | 解决 `InviteService::GetInviterByCode()` 全表扫描问题，改为索引查询 |
| 影响模块 | shopxo-backend/app/service/InviteService.php、sxo_user 表、muying-final-migration.sql |
| 预估工作量 | 1 天 |
| 依赖关系 | 无 |
| 具体操作 | 1. `sxo_user` 表新增 `invite_code` char(32) 字段并添加唯一索引；2. 编写存量用户邀请码生成脚本；3. 修改 InviteService::GetInviterByCode() 改为 `WHERE invite_code = ?`；4. 用户注册时自动生成邀请码存储；5. InviteInfo 中直接读取 invite_code 字段而非实时计算 |
| 验收标准 | 邀请码反查走索引，注册接口响应时间 < 200ms（万级用户量） |

## P1 — 高优先级（上线后 1-2 周）

### P1-01: 分享海报 Canvas 生成

| 项目 | 内容 |
|------|------|
| 功能描述 | 使用 Canvas 绘制活动海报和邀请海报，用户可保存到相册分享 |
| 影响模块 | pages/activity-detail/activity-detail.vue、pages/invite/invite.vue、后端 Poster 接口 |
| 预估工作量 | 3 天 |
| 依赖关系 | 无 |
| 具体操作 | 1. 设计海报模板（活动海报含封面/标题/时间/地址/小程序码；邀请海报含头像/昵称/邀请码/小程序码）；2. 前端使用 uni-app Canvas API 绘制；3. 生成后调用 `uni.saveImageToPhotosAlbum()` 保存；4. 需处理小程序码生成（后端调用微信接口或使用现成二维码） |
| 验收标准 | 用户可在活动详情页和邀请页生成海报并保存到相册 |

### P1-02: 优惠券发放对接

| 项目 | 内容 |
|------|------|
| 功能描述 | 实现 `reward_type=coupon` 的邀请奖励发放逻辑，支持邀请注册/首单发放优惠券 |
| 影响模块 | app/service/InviteService.php、sxo_invite_reward 表、后台奖励配置 |
| 预估工作量 | 2 天 |
| 依赖关系 | ShopXO 优惠券插件已安装 |
| 具体操作 | 1. InviteService::GrantReward() 新增 coupon 分支；2. 调用 ShopXO 原生优惠券发放 API；3. 后台配置项增加优惠券 ID 选择；4. 测试注册/首单触发优惠券发放 |
| 验收标准 | 配置优惠券奖励后，被邀请人注册/首单时邀请人自动收到优惠券 |

### P1-03: 收藏/点赞后端 API

| 项目 | 内容 |
|------|------|
| 功能描述 | 实现活动收藏和点赞的后端 API，使前端状态可持久化和跨设备同步 |
| 影响模块 | 新增 ActivityFavorService 或复用 GoodsFavorService、sxo_activity 表（新增 favor_count/like_count）、前端 activity-detail.vue |
| 预估工作量 | 2 天 |
| 依赖关系 | 无 |
| 具体操作 | 1. 新增 `sxo_activity_favor` 表（user_id, activity_id, add_time）；2. 新增收藏 API（favor/cancel/is_favored）；3. 新增点赞 API（like/cancel/is_liked）；4. 活动表新增 favor_count/like_count 计数字段；5. 前端对接 API 替换本地状态 |
| 验收标准 | 收藏/点赞状态持久化，不同设备登录后状态一致，活动列表显示收藏/点赞数 |

### P1-04: 评论功能
