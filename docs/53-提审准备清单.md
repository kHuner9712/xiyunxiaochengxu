# 提审准备清单

> 按微信小程序上线要求，整理需要人工准备的内容。
> 每项标注：✅已准备 / ❌未准备 / ⚠️部分准备

---

## 一、域名与 HTTPS

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 正式域名 | 微信小程序强制要求 HTTPS | ❌ 使用 localhost | 购买域名（如 muying.com） |
| SSL 证书 | 域名对应的 HTTPS 证书 | ❌ 无 | 在云服务商申请免费/付费 SSL 证书 |
| 服务器部署 | 公网可访问的后端服务 | ❌ 本地 Docker | 购买云服务器（2核4G起步），部署 Docker 环境 |
| request 合法域名 | 微信公众平台配置 | ❌ 未配置 | 在小程序后台「开发→开发设置」添加 https://你的域名/ |

**操作步骤**：
1. 购买域名 + 云服务器
2. 域名备案（国内服务器必须，约7-20天）
3. 配置 DNS 解析到服务器 IP
4. 申请 SSL 证书并配置到 Nginx
5. 修改 App.vue 中 request_url 为 https://你的域名/
6. 在微信公众平台添加 request 合法域名

## 二、微信小程序 AppID

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 小程序注册 | 微信公众平台注册 | ❌ 未注册 | 前往 mp.weixin.qq.com 注册 |
| AppID | 小程序唯一标识 | ❌ 使用测试号 | 注册后获取 |
| AppSecret | 小程序密钥 | ❌ 无 | 注册后在后台获取 |
| 修改 manifest.json | 填入真实 AppID | ❌ 未修改 | 获取 AppID 后修改 |

**操作步骤**：
1. 用企业/个体工商户营业执照注册微信小程序
2. 获取 AppID 和 AppSecret
3. 在后台「系统设置」配置 AppID 和 AppSecret
4. 修改 shopxo-uniapp/manifest.json 中的 appid 字段

## 三、隐私政策与用户协议

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 隐私政策 | 微信强制要求，需在小程序内可访问 | ❌ 无 | 撰写隐私政策页面 |
| 用户协议 | 建议提供 | ❌ 无 | 撰写用户服务协议页面 |
| 隐私弹窗 | 首次使用需弹窗告知 | ❌ 无 | 添加隐私弹窗组件 |

**隐私政策必须包含**：
- 收集的个人信息类型（手机号、阶段信息、预产期等）
- 信息使用目的
- 信息存储方式和期限
- 用户权利（查询、删除、撤回同意）
- 第三方共享说明
- 联系方式

**操作步骤**：
1. 参考模板撰写隐私政策
2. 在小程序中添加隐私政策页面
3. 在微信公众平台填写隐私保护指引
4. 添加首次使用隐私弹窗

## 四、客服方式

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 在线客服 | 微信要求小程序提供客服入口 | ❌ 无 | 添加客服按钮 |

**操作步骤**：
1. 在小程序后台「客服」模块添加客服人员
2. 在小程序「我的」页面添加客服入口按钮（使用 button open-type="contact"）

## 五、资质与类目

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 营业执照 | 企业或个体工商户 | ❌ 待确认 | 准备营业执照扫描件 |

## 六、支付配置

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 微信支付商户号 | 需单独申请 | ❌ 无 | 前往 pay.weixin.qq.com 申请 |
| 支付接口对接 | 商户号+API密钥 | ❌ 未对接 | 在 ShopXO 后台配置支付参数 |
| 首单奖励触发 | 支付成功回调中调用 | ⚠️ 代码已写但未接入真实支付 | 配置支付后验证 |

**操作步骤**：
1. 用营业执照申请微信支付商户号
2. 绑定小程序 AppID
3. 获取商户号(mch_id)、API密钥、证书
4. 在 ShopXO 后台「支付管理」配置微信支付
5. 测试支付流程
6. 验证首单奖励是否在支付成功后触发

## 七、服务器与数据库

| 项目 | 要求 | 当前状态 | 需要做什么 |
|------|------|----------|-----------|
| 云服务器 | 公网可访问 | ❌ 本地 Docker | 购买云服务器 |
| MySQL | 生产级数据库 | ❌ Docker 本地 | 云服务器上部署 MySQL 或使用云数据库 |
| 域名备案 | 国内服务器必须 | ❌ 未备案 | 域名+服务器一起备案 |
| 数据库密码 | 生产环境不能使用 root123456 | ❌ 弱密码 | 修改为强密码 |
| Redis 缓存 | 生产环境建议使用 | ❌ 未配置 | 安装 Redis 并配置 |

**建议配置**：
- 服务器：2核4G（阿里云/腾讯云轻量应用服务器）
- 系统：Ubuntu 22.04 / CentOS 8
- 数据库：MySQL 5.7+（推荐 8.0），强密码，禁止 root 远程登录
- PHP：8.1+，OPcache 开启

## 九、提审流程

1. 完成上述所有准备工作
2. 在微信开发者工具中点击「上传」
3. 登录微信公众平台，进入「版本管理」
4. 将开发版提交审核
5. 填写审核信息（功能页面、类目、测试账号等）
6. 等待审核（通常1-7天）
7. 审核通过后点击「发布」
