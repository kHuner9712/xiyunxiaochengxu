# 禧孕母婴用品私域商城小程序 - 会员与营销系统

## 1. 文档概述

### 1.1 文档目的

本文档详细定义"禧孕母婴用品私域商城小程序"的会员与营销系统设计方案，涵盖会员等级体系、积分系统、复购系统、活动转化系统以及营销活动优先级与互斥规则等核心模块。本文档将作为前后端开发、测试验收及运营配置的依据。

### 1.2 适用范围

- 产品经理：需求确认与功能验收
- 前端开发：小程序端页面与交互实现
- 后端开发：接口设计与业务逻辑实现
- 运营人员：后台配置与活动运营
- 测试人员：功能测试与回归测试

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| 成长值 | 衡量会员等级的数值，通过消费及互动行为获取 |
| 积分 | 平台虚拟货币，可抵扣订单金额 |
| GMV | Gross Merchandise Volume，商品交易总额 |
| ROI | Return on Investment，投资回报率 |
| 私域流量 | 品牌自有可反复触达的用户流量 |
| 复购率 | 在一定周期内重复购买的用户占比 |
| 客单价 | 平均每笔订单金额 |
| 转化率 | 从浏览到下单的转化比例 |

### 1.4 系统定位

本系统为禧孕文化传媒有限公司自营商城的会员与营销系统，非多商户平台。所有营销活动、会员权益、积分规则均由平台统一管理，不存在商户级别的独立配置。

## 2. 会员系统

### 2.1 会员等级体系

#### 2.1.1 等级总览

| 属性 | 普通会员 | 银卡会员 | 金卡会员 | 黑金会员 |
|------|----------|----------|----------|----------|
| 升级条件 | 注册即成为 | 成长值≥1000 | 成长值≥5000 | 成长值≥20000 |
| 成长值范围 | 0-999 | 1000-4999 | 5000-19999 | 20000+ |
| 等级图标 | 铜色 | 银色 | 金色 | 黑金色 |
| 等级折扣 | 无折扣 | 98折 | 95折 | 9折 |
| 积分倍率 | 1.0x | 2.0x | 3.0x | 5.0x |
| 等级标识 | 🥉 | 🥈 | 🥇 | 💎 |

#### 2.1.2 普通会员

- **升级条件**：用户完成注册即自动成为普通会员，无需额外操作
- **成长值范围**：0-999
- **等级图标**：铜色徽章，展示在用户头像旁及会员中心
- **等级折扣**：无折扣，按商品原价购买
- **积分倍率**：1.0x，即1元=1基础积分
- **可享权益**：
  - 基础积分获取
  - 参与平台公开活动
  - 新人礼包领取
  - 基础客服服务
- **设计意图**：作为入门等级，引导用户了解会员体系，激励消费升级

#### 2.1.3 银卡会员

- **升级条件**：成长值达到1000
- **成长值范围**：1000-4999
- **等级图标**：银色徽章
- **等级折扣**：98折，即商品价格×0.98
- **积分倍率**：2.0x，即1元=2基础积分
- **可享权益**：
  - 98折购物折扣
  - 双倍积分获取
  - 银卡专属优惠券（每月2张）
  - 生日礼（生日当月优惠券礼包）
  - 宝宝生日提醒与专属优惠
  - 参与银卡及以上会员专属活动
- **设计意图**：培养用户消费习惯，让用户初步感知会员等级价值

#### 2.1.4 金卡会员

- **升级条件**：成长值达到5000
- **成长值范围**：5000-19999
- **等级图标**：金色徽章
- **等级折扣**：95折，即商品价格×0.95
- **积分倍率**：3.0x，即1元=3基础积分
- **可享权益**：
  - 95折购物折扣
  - 三倍积分获取
  - 金卡专属优惠券（每月4张）
  - 生日礼（生日当月优惠券礼包+额外积分）
  - 宝宝生日提醒与专属优惠
  - 参与金卡及以上会员专属活动
  - 优先参与限量活动
  - 专属活动优先通知
- **设计意图**：作为核心消费用户等级，提供显著差异化权益，增强用户粘性

#### 2.1.5 黑金会员

- **升级条件**：成长值达到20000
- **成长值范围**：20000及以上
- **等级图标**：黑金色徽章（带动态光效）
- **等级折扣**：9折，即商品价格×0.9
- **积分倍率**：5.0x，即1元=5基础积分
- **可享权益**：
  - 9折购物折扣
  - 五倍积分获取
  - 黑金专属优惠券（每月6张，面额更高）
  - 生日礼（生日当月优惠券礼包+额外积分+生日专属礼品）
  - 宝宝生日提醒与专属优惠
  - 黑金会员专属活动
  - 专属客服通道（1对1客服，响应时间≤5分钟）
  - 订单优先发货（标记优先处理）
  - 新品优先购买权
  - 年度黑金礼盒
  - 线下活动优先邀请
- **设计意图**：顶级会员等级，提供全方位尊享服务，打造品牌忠诚度和口碑传播

### 2.2 会员权益详细设计

#### 2.2.1 等级折扣

| 规则项 | 详细说明 |
|--------|----------|
| 折扣类型 | 商品价格直接打折 |
| 折扣基准 | 以商品售价为基准（非原价） |
| 折扣叠加 | 等级折扣与限时折扣不叠加，取优惠力度更大者 |
| 折扣与优惠券 | 等级折扣后可再使用优惠券 |
| 折扣与满减 | 等级折扣后金额参与满减计算 |
| 折扣展示 | 商品详情页展示"会员价"标签，显示折后价 |
| 折扣范围 | 全场商品均参与等级折扣（特殊商品除外，如已特价商品） |
| 不参与折扣商品 | 后台可配置不参与等级折扣的商品，商品详情页标注"不参与会员折扣" |

**实现方式**：

- 后端接口返回商品信息时，根据用户等级计算会员价一并返回
- 前端商品列表、商品详情页均展示会员价
- 下单时后端重新校验等级折扣，防止前端篡改
- 等级折扣计算公式：`会员价 = 商品售价 × 等级折扣率`

#### 2.2.2 专属优惠券

| 等级 | 每月可领数量 | 优惠券类型 | 典型面额 | 领取时间 |
|------|-------------|-----------|---------|---------|
| 普通会员 | 0 | - | - | - |
| 银卡会员 | 2 | 满减券 | 满100减8、满200减15 | 每月1日自动发放 |
| 金卡会员 | 4 | 满减券+品类券 | 满100减12、满200减25、母婴品类券满150减20 | 每月1日自动发放 |
| 黑金会员 | 6 | 满减券+品类券+无门槛券 | 满100减18、满200减35、母婴品类券满150减30、无门槛10元券×2 | 每月1日自动发放 |

**实现方式**：

- 后台配置各等级优惠券模板，包含面额、使用门槛、有效期等
- 每月1日通过定时任务自动发放对应等级优惠券到用户账户
- 用户升级时，当月未领取的等级优惠券立即补发
- 用户降级时，已发放的优惠券不回收，但下次发放按新等级执行
- 优惠券有效期：发放后30天有效
- 优惠券领取记录存入 `user_coupon` 表

#### 2.2.3 生日礼

| 规则项 | 详细说明 |
|--------|----------|
| 触发条件 | 用户生日当月（根据注册时填写的生日信息） |

| 领取方式 | 生日当月首次登录时弹窗提醒，用户主动领取 |
| 有效期 | 领取后15天有效 |
| 每年限额 | 每年1次，生日月份未领取不补发 |
| 未填写生日 | 不触发生日礼，引导用户完善生日信息 |

#### 2.2.4 宝宝生日提醒

| 提醒时间 | 宝宝生日前7天推送提醒消息 |

| 多宝宝情况 | 每个宝宝生日独立计算，均可享受 |

**实现方式**：

#### 2.2.5 积分倍率

| 等级 | 积分倍率 | 说明 |
|------|---------|------|
| 普通会员 | 1.0x | 1元消费=1基础积分 |
| 银卡会员 | 2.0x | 1元消费=2基础积分 |
| 金卡会员 | 3.0x | 1元消费=3基础积分 |
| 黑金会员 | 5.0x | 1元消费=5基础积分 |

**积分计算公式**：`获得积分 = floor(实付金额 × 积分倍率)`

**实现方式**：

- 订单完成后（确认收货），根据用户当前等级的积分倍率计算积分
- 积分倍率以订单完成时的用户等级为准，非下单时
- 积分倍率与签到、分享等其他积分获取方式无关，仅适用于消费积分

### 2.3 成长值体系

#### 2.3.1 成长值获取方式

| 行为 | 成长值 | 说明 | 上限 |
|------|--------|------|------|
| 消费1元 | 1成长值 | 按实付金额计算，不含积分抵扣部分 | 无上限 |
| 每日签到 | 5成长值 | 每日签到获得 | 每日1次 |
| 连续签到7天 | 额外20成长值 | 连续签到7天额外奖励 | 每周期1次 |
| 完善个人资料 | 20成长值 | 一次性奖励 | 一次性 |
| 完善宝宝档案 | 20成长值/个 | 每个宝宝档案完善奖励 | 最多3个 |
| 首次下单 | 50成长值 | 首次完成订单奖励 | 一次性 |
| 评价订单 | 10成长值/条 | 订单评价奖励 | 每日最多3条 |
| 分享商品 | 3成长值/次 | 分享商品到微信好友/朋友圈 | 每日最多3次 |
| 参与活动 | 5-50成长值 | 根据活动类型不同 | 按活动规则 |

#### 2.3.2 成长值与消费金额的关系

```
消费获得成长值 = floor(实付金额 - 积分抵扣金额)
```

**说明**：

- 实付金额指用户实际支付的金额（含优惠券抵扣，不含积分抵扣）
- 积分抵扣部分不计算成长值，避免"积分换成长值→成长值升级→更多积分"的循环套利
- 退款订单扣除对应成长值
- 部分退款按退款比例扣除成长值

**示例**：

| 场景 | 商品金额 | 优惠券抵扣 | 积分抵扣 | 实付金额 | 获得成长值 |
|------|---------|-----------|---------|---------|-----------|
| 全额支付 | 100元 | 0 | 0 | 100元 | 100 |
| 使用优惠券 | 100元 | 10元 | 0 | 90元 | 90 |
| 使用积分抵扣 | 100元 | 0 | 30元 | 70元 | 70 |
| 优惠券+积分 | 100元 | 10元 | 20元 | 70元 | 70 |

#### 2.3.3 成长值计算逻辑

```
用户总成长值 = 历史累计成长值 - 降级扣除成长值
```

- 成长值为累计值，不设过期
- 退款时扣除对应成长值，若扣除后低于当前等级下限，不立即降级，在降级考核期处理
- 成长值变动实时记录，存入 `growth_value_log` 表

**数据表设计**：

```
growth_value_log
├── id (主键)
├── user_id (用户ID)
├── change_value (变动值，正数增加，负数减少)
├── current_value (变动后当前值)
├── source (来源：order/sign_in/profile/baby_profile/first_order/review/share/activity)
├── source_id (来源ID，如订单ID)
├── created_at (创建时间)
```

#### 2.3.4 等级升级判定时机

| 判定时机 | 说明 |
|----------|------|
| 订单完成后 | 用户确认收货时，计算消费成长值并判定是否升级 |
| 签到后 | 每日签到后判定 |
| 评价后 | 订单评价后判定 |
| 资料完善后 | 完善个人资料/宝宝档案后判定 |
| 定时任务 | 每日凌晨2:00全量扫描，处理遗漏情况 |

**升级流程**：

1. 触发成长值变动
2. 计算用户当前总成长值
3. 判断是否达到更高等级阈值
4. 若达到，执行升级操作：
   - 更新用户等级
   - 发放升级奖励（优惠券+积分）
   - 推送升级通知
   - 记录升级日志
5. 升级奖励内容：

| 升级方向 | 升级奖励 |
|----------|---------|
| 普通→银卡 | 50积分 + 满100减10优惠券×1 |
| 银卡→金卡 | 200积分 + 满100减15优惠券×2 |
| 金卡→黑金 | 500积分 + 满100减20优惠券×3 + 黑金专属礼品 |

#### 2.3.5 等级降级规则

| 规则项 | 详细说明 |
|--------|----------|
| 考核周期 | 每6个月考核1次（1月1日、7月1日） |
| 考核条件 | 考核期内成长值净增长是否达到当前等级最低要求 |
| 降级条件 | 考核期内成长值净增长 < 当前等级最低成长值 × 20% |
| 降级幅度 | 降一级（如金卡→银卡，不会跨级降级） |
| 降级通知 | 考核日前15天推送预警通知 |
| 降级执行 | 考核日当天执行降级 |
| 普通会员 | 不参与降级考核，始终为普通会员 |

**各等级考核标准**：

| 等级 | 考核期需净增成长值 | 说明 |
|------|-------------------|------|
| 银卡会员 | ≥200 | 半年内需净增200成长值（约消费200元） |
| 金卡会员 | ≥1000 | 半年内需净增1000成长值（约消费1000元） |
| 黑金会员 | ≥4000 | 半年内需净增4000成长值（约消费4000元） |

**降级预警通知**：

## 3. 积分系统

### 3.1 积分获取

#### 3.1.1 下单得积分

| 规则项 | 详细说明 |
|--------|----------|
| 计算公式 | `获得积分 = floor(实付金额 × 积分倍率)` |
| 基础倍率 | 1元=1基础积分 |
| 积分倍率 | 根据会员等级不同：普通1.0x、银卡2.0x、金卡3.0x、黑金5.0x |
| 发放时机 | 订单确认收货后发放 |
| 积分抵扣部分 | 积分抵扣的金额部分不参与积分返还计算 |
| 退款处理 | 退款时扣除对应积分（按获得时倍率扣除） |
| 部分退款 | 按退款金额比例扣除积分 |

**计算公式（含积分抵扣）**：

```
可参与返积分金额 = 实付金额 - 积分抵扣金额
获得积分 = floor(可参与返积分金额 × 会员积分倍率)
```

**示例**：

| 场景 | 订单金额 | 优惠券 | 积分抵扣 | 实付 | 会员等级 | 积分倍率 | 获得积分 |
|------|---------|--------|---------|------|---------|---------|---------|
| A | 200元 | 0 | 0 | 200元 | 普通 | 1.0x | 200 |
| B | 200元 | 0 | 0 | 200元 | 金卡 | 3.0x | 600 |
| C | 200元 | 20元 | 0 | 180元 | 银卡 | 2.0x | 360 |
| D | 200元 | 0 | 60元 | 140元 | 金卡 | 3.0x | floor((140-60)×3.0) = 240 |
| E | 200元 | 20元 | 40元 | 140元 | 黑金 | 5.0x | floor((140-40)×5.0) = 500 |

### 3.2 积分抵扣

#### 3.2.1 抵扣规则

| 规则项 | 详细说明 |
|--------|----------|
| 抵扣比例 | 100积分=1元 |
| 最低使用量 | 单笔订单最低使用100积分起抵（即最低抵扣1元） |
| 最高抵扣比例 | 单笔订单最高抵扣订单金额的30% |
| 抵扣基准 | 以优惠券、活动折扣后的金额为基准计算30%上限 |
| 积分抵扣部分 | 不参与积分返还、不计算成长值 |
| 不可抵扣商品 | 后台可配置不参与积分抵扣的商品 |
| 不可抵扣场景 | 运费不参与积分抵扣 |

#### 3.2.2 积分抵扣计算

```
步骤1：计算订单应付金额 = 商品金额 - 优惠折扣 - 优惠券抵扣
步骤2：计算最大可抵扣金额 = 订单应付金额 × 30%
步骤3：计算用户可抵扣金额 = min(用户积分 / 100, 最大可抵扣金额)
步骤4：最终实付金额 = 订单应付金额 - 积分抵扣金额
```

**示例**：

| 场景 | 商品金额 | 优惠折扣 | 优惠券 | 应付金额 | 30%上限 | 用户积分 | 积分抵扣 | 实付金额 |
|------|---------|---------|--------|---------|---------|---------|---------|---------|
| A | 200元 | 0 | 0 | 200元 | 60元 | 3000 | 30元 | 170元 |
| B | 200元 | 0 | 0 | 200元 | 60元 | 10000 | 60元 | 140元 |
| C | 200元 | 20元 | 10元 | 170元 | 51元 | 8000 | 51元 | 119元 |
| D | 100元 | 0 | 0 | 100元 | 30元 | 50 | 0（不足100积分） | 100元 |

### 3.4 积分计算示例

#### 示例1：普通会员日常购物

```
用户等级：普通会员（积分倍率1.0x）
商品：婴儿奶粉 298元
优惠券：满200减20
积分抵扣：用户有500积分，选择使用

计算过程：
1. 商品金额：298元
2. 优惠券抵扣：20元
3. 应付金额：298 - 20 = 278元
4. 最大积分抵扣：278 × 30% = 83.4元
5. 用户可抵扣：min(500/100, 83.4) = min(5, 83.4) = 5元
6. 实付金额：278 - 5 = 273元
7. 可返积分金额：273 - 5 = 268元（积分抵扣部分不返积分）
8. 获得积分：floor(268 × 1.0) = 268积分
```

#### 示例2：金卡会员大额购物

```
用户等级：金卡会员（积分倍率3.0x，95折）
商品：婴儿推车 1299元
等级折扣：1299 × 0.95 = 1234.05元
优惠券：满1000减50
积分抵扣：用户有8000积分，选择使用

计算过程：
1. 商品金额：1299元
2. 等级折扣：1299 × 0.95 = 1234.05元
3. 优惠券抵扣：50元
4. 应付金额：1234.05 - 50 = 1184.05元
5. 最大积分抵扣：1184.05 × 30% = 355.22元
6. 用户可抵扣：min(8000/100, 355.22) = min(80, 355.22) = 80元
7. 实付金额：1184.05 - 80 = 1104.05元
8. 可返积分金额：1104.05 - 80 = 1024.05元
9. 获得积分：floor(1024.05 × 3.0) = 3072积分
10. 获得成长值：floor(1024.05) = 1024成长值
```

#### 示例3：黑金会员组合购买

```
用户等级：黑金会员（积分倍率5.0x，9折）
商品A：纸尿裤 189元
商品B：湿巾 39元
商品C：奶瓶 89元
合计：317元
等级折扣：317 × 0.9 = 285.3元
满减活动：满200减25
优惠券：满100减15
积分抵扣：用户有2000积分，选择使用

计算过程：
1. 商品合计：317元
2. 等级折扣：317 × 0.9 = 285.3元
3. 满减抵扣：25元（285.3 > 200，满足满200减25）
4. 优惠券抵扣：15元
5. 应付金额：285.3 - 25 - 15 = 245.3元
6. 最大积分抵扣：245.3 × 30% = 73.59元
7. 用户可抵扣：min(2000/100, 73.59) = min(20, 73.59) = 20元
8. 实付金额：245.3 - 20 = 225.3元
9. 可返积分金额：225.3 - 20 = 205.3元
10. 获得积分：floor(205.3 × 5.0) = 1026积分
11. 获得成长值：floor(205.3) = 205成长值
```

| 纸尿裤 | 箱装（160片） | 1.5 | 38天 |

#### 4.1.3 用户购买后自动创建提醒

- 用户购买高复购商品并确认收货后，系统自动创建复购提醒

- 同一商品同一用户仅保留1条有效提醒
- 用户再次购买同一商品时，重置提醒周期

**数据表设计**：

```

├── id (主键)

├── base_cycle (基础周期天数)
├── custom_cycle (用户自定义周期天数，NULL表示使用默认)
├── remind_at (提醒时间)
├── status (状态：active/reminded/cancelled)
├── created_at (创建时间)
├── updated_at (更新时间)
```

### 4.2 基于购买记录的复购推荐

| 购买频率 | 用户标签 | 推荐策略 |

| 月购买≥3次 | 高频用户 | 推荐大包装/箱装，强调性价比 |
| 月购买1-2次 | 中频用户 | 推荐常规规格+关联商品 |
| 月购买<1次 | 低频用户 | 推荐小包装/试用装，降低决策门槛 |

#### 4.2.3 库存不足提醒（复购商品）

- 用户常购商品库存低于安全值时，推送库存紧张提醒
- 安全值定义：库存 < 近30天日均销量 × 7
- 提醒内容："您常买的{商品名}库存紧张，建议尽快补货"
- 商品详情页展示"库存紧张"标签

### 4.3 基于宝宝月龄的推荐

#### 4.3.1 月龄阶段划分

| 阶段 | 月龄范围 | 关键需求 | 典型商品 |
|------|---------|---------|---------|
| 新生儿 | 0-3个月 | 喂养、换洗、睡眠 | 奶粉1段、NB/S码纸尿裤、婴儿床品 |
| 婴儿早期 | 4-6个月 | 辅食添加、出牙准备 | 米粉、咬胶、围嘴 |
| 婴儿中期 | 7-12个月 | 辅食丰富、学爬学走 | 辅食工具、爬行垫、学步鞋 |
| 幼儿早期 | 13-18个月 | 学步、语言启蒙 | 学步鞋、绘本、积木 |
| 幼儿期 | 19-36个月 | 如厕训练、早教 | 训练裤、早教玩具、绘本 |
| 学龄前期 | 37-72个月 | 独立性培养、社交 | 儿童洗护、学习用品 |

#### 4.3.2 适龄商品匹配

- 每个商品在后台配置适用月龄范围（如：奶粉1段 0-6个月）
- 商品可配置多月龄段标签
- 根据宝宝当前月龄自动匹配适龄商品
- 多宝宝情况下，分别推荐各宝宝适龄商品

**数据表设计**：

```
product_age_tag
├── id (主键)
├── product_id (商品ID)
├── age_stage (月龄阶段编码)
├── sort_order (排序权重)
├── created_at (创建时间)
```

#### 4.3.3 推荐展示位置

| 位置 | 展示方式 | 说明 |
|------|---------|------|
| 首页 | "宝宝适龄推荐"模块 | 根据宝宝月龄展示6-8个商品 |
| 商品详情页 | "宝宝也适合"模块 | 展示与当前商品同月龄段的其他商品 |
| 搜索结果页 | 月龄筛选标签 | 支持按宝宝月龄筛选搜索结果 |
| 消息中心 | 月龄变化推送 | 宝宝进入新月龄阶段时推送适龄商品推荐 |

#### 4.3.4 推荐算法逻辑

```
1. 获取用户宝宝档案中的出生日期
2. 计算宝宝当前月龄 = floor((当前日期 - 出生日期) / 30)
3. 查询适用当前月龄段的商品集合
4. 排序规则：
   a. 优先级1：用户历史购买品牌的商品（品牌偏好）
   b. 优先级2：用户浏览过但未购买的商品（兴趣偏好）
   c. 优先级3：销量高的商品（热门偏好）
   d. 优先级4：利润率高的商品（商业偏好）
5. 去重：排除用户近期（30天内）已购买的商品
6. 返回推荐列表（最多20个商品）
```

### 4.4 基于历史订单的关联推荐

#### 4.4.1 购买了A商品的客户也购买了B商品

**关联规则挖掘**：

- 基于全平台订单数据，使用关联规则算法（Apriori）挖掘商品关联关系
- 计算指标：支持度（Support）、置信度（Confidence）、提升度（Lift）
- 筛选条件：支持度≥0.01，置信度≥0.3，提升度≥1.5
- 更新频率：每周日凌晨全量更新

**展示规则**：

| 展示位置 | 展示方式 | 商品数量 |
|----------|---------|---------|
| 商品详情页 | "买了又买"模块 | 6个商品 |
| 购物车 | "搭配购买"模块 | 4个商品 |
| 订单完成页 | "猜你还需要"模块 | 4个商品 |

#### 4.4.2 同分类商品推荐

- 在商品详情页展示"同类好物"模块
- 展示同分类下销量Top商品
- 排除当前商品
- 最多展示8个商品

#### 4.4.3 品牌关联推荐

- 用户购买某品牌商品后，推荐同品牌其他商品
- 在商品详情页展示"同品牌推荐"模块
- 最多展示6个商品
- 排序：新品优先 → 销量优先

**互斥规则说明**：

| 互斥组合 | 处理方式 | 原因 |
|----------|---------|------|
| 会员折扣 vs 限时折扣 | 取价格更低者 | 避免双重折扣导致价格过低 |
| 会员折扣 vs 组合套餐 | 组合套餐优先，不叠加会员折扣 | 套餐已是优惠价 |
| 限时折扣 vs 组合套餐 | 组合套餐优先，不叠加限时折扣 | 套餐已是优惠价 |
| 满减 vs 满减 | 同一时间仅1个满减活动生效 | 避免满减叠加 |
| 满赠 vs 满赠 | 同一时间仅1个满赠活动生效 | 避免赠品叠加 |
| 优惠券 vs 优惠券 | 单笔订单仅使用1张优惠券 | 避免券叠加 |
| 组合套餐 vs 其他优惠 | 套餐不参与任何其他优惠 | 套餐为独立优惠体系 |

### 6.2 最优优惠计算逻辑

当用户购物车中商品同时满足多个优惠条件时，系统自动计算最优组合：

```
最优优惠计算流程：

1. 确定商品基础价格
   - 若商品参与限时折扣：基础价 = min(会员价, 限时折扣价)
   - 若商品参与组合套餐：基础价 = 套餐价（套餐商品不参与后续折扣计算）
   - 其他：基础价 = 会员价

2. 计算满减优惠
   - 汇总参与满减活动的商品金额
   - 匹配最高满足的满减阶梯
   - 满减金额 = 匹配阶梯的优惠金额

3. 计算优惠券优惠
   - 筛选用户可用优惠券列表
   - 逐一计算每张优惠券可抵扣金额
   - 选择抵扣金额最大的优惠券

4. 计算积分抵扣
   - 最大可抵扣金额 = (应付金额 - 满减 - 优惠券) × 30%
   - 实际可抵扣 = min(用户积分/100, 最大可抵扣金额)

5. 计算满赠
   - 判断是否满足满赠条件
   - 满足则添加赠品

6. 组合计算
   - 遍历所有合法优惠组合
   - 计算每种组合的最终实付金额
   - 选择实付金额最低的组合

7. 推荐给用户
   - 默认选择最优组合
   - 用户可手动调整（如更换优惠券、调整积分使用量）
```

**最优组合示例**：

```
场景：金卡会员购买商品
商品金额：300元
会员折扣：95折 → 285元
可用满减：满200减25
可用优惠券：满100减15、满200减30
用户积分：5000积分

组合1：会员折扣 + 满减 + 满100减15券 + 积分
  = 285 - 25 - 15 - min(50, (285-25-15)×30%)
  = 285 - 25 - 15 - min(50, 73.5)
  = 285 - 25 - 15 - 50
  = 195元

组合2：会员折扣 + 满减 + 满200减30券 + 积分
  = 285 - 25 - 30 - min(50, (285-25-30)×30%)
  = 285 - 25 - 30 - min(50, 69)
  = 285 - 25 - 30 - 50
  = 180元

最优组合：组合2，实付180元
```

### 6.3 价格计算流程图

```
商品原价
    │
    ▼
┌─────────────────────────┐
│ 是否为组合套餐商品？      │
│ 是 → 使用套餐价格        │
│ 否 → 继续以下流程        │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│ 是否参与限时折扣？        │
│ 是 → 计算限时折扣价      │
│ 否 → 使用商品售价        │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│ 会员等级折扣 vs 限时折扣  │
│ 取价格更低者作为商品基价  │
└─────────────────────────┘
    │
    ▼
  商品基价（汇总所有商品）
    │
    ▼
┌─────────────────────────┐
│ 计算满减优惠             │
│ 匹配满减阶梯，扣减金额   │
└─────────────────────────┘
    │
    ▼
  满减后金额
    │
    ▼
┌─────────────────────────┐
│ 计算优惠券抵扣           │
│ 选择最优优惠券，扣减金额  │
└─────────────────────────┘
    │
    ▼
  优惠券后金额
    │
    ▼
┌─────────────────────────┐
│ 计算积分抵扣             │
│ 最多抵扣30%，最低100积分 │
└─────────────────────────┘
    │
    ▼
  积分抵扣后金额
    │
    ▼
┌─────────────────────────┐
│ 加上运费                 │
└─────────────────────────┘
    │
    ▼
  最终实付金额
    │
    ▼
┌─────────────────────────┐
│ 判断满赠条件             │
│ 满足则添加赠品           │
└─────────────────────────┘
    │
    ▼
  订单确认（含赠品信息）
```

**价格计算公式汇总**：

```
1. 商品基价 = min(会员价, 限时折扣价)  或  套餐价
2. 商品基价合计 = Σ(各商品基价 × 数量)
3. 满减金额 = 匹配满减阶梯优惠
4. 优惠券金额 = 最优优惠券抵扣额
5. 积分抵扣金额 = min(用户积分/100, (商品基价合计-满减-优惠券)×30%)
6. 商品应付 = 商品基价合计 - 满减 - 优惠券 - 积分抵扣
7. 运费 = 计算运费（满额免运费等规则）
8. 实付金额 = 商品应付 + 运费
```
