# 部署与发布说明

## 一、环境要求

### 1.1 必备软件

| 软件           | 最低版本 | 推荐版本   | 说明                       |
| -------------- | -------- | ---------- | -------------------------- |
| Docker Desktop | 20.10+   | 24.0+      | Windows 版需开启 WSL2 后端 |
| Node.js        | 14.x     | 18.x LTS   | 前端编译依赖               |
| HBuilderX      | 3.6+     | 最新版     | uni-app 编译运行（必须）   |
| 微信开发者工具 | 1.06+    | 最新稳定版 | 小程序预览与调试           |
| Git            | 2.30+    | 最新版     | 代码版本管理               |

### 1.2 Docker 镜像版本

| 镜像  | 版本                | 说明                                        |
| ----- | ------------------- | ------------------------------------------- |
| nginx | 1.25-alpine         | Web 服务器                                  |
| php   | 8.1-fpm-alpine      | PHP-FPM 应用服务（自构建，含扩展）          |
| mysql | **8.0**（当前仓库） | 本地开发使用 8.0；预部署目标为 MySQL 5.7.44 |

> **重要区分**：
>
> - **当前仓库本地开发**：`docker-compose.yml` 使用 `mysql:8.0`，默认端口映射 `127.0.0.1:13306->3306`

> - **SQL 兼容性**：`docs/muying-final-migration.sql` B 段使用 `information_schema` 检查，兼容 5.7

### 1.3 PHP 扩展（Dockerfile 已内置）

pdo_mysql, mysqli, gd, mbstring, zip, bcmath, fileinfo, pcntl, opcache, redis

### 1.4 系统要求

- Windows 10/11 64位
- 内存 ≥ 8GB（Docker 运行需要）
- 磁盘空间 ≥ 10GB（Docker 镜像 + 数据）
- 网络环境：需能访问 Docker Hub、npm 仓库

## 二、本地开发环境启动步骤

### 2.1 Docker 启动

```powershell
cd c:\Users\15864\Desktop\yunxixiaochengxu
docker compose up -d
```

首次启动会自动构建 PHP 镜像并拉取 Nginx/MySQL 镜像，约需 5-10 分钟。

验证容器状态：

```powershell
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

应看到三个容器均为 `Up` 状态：

| 容器名       | 端口映射                  | 说明             |
| ------------ | ------------------------- | ---------------- |
| shopxo-nginx | 127.0.0.1:8080->80/tcp    | Web 服务器       |
| shopxo-php   | 9000/tcp                  | PHP-FPM 应用服务 |
| shopxo-mysql | 127.0.0.1:13306->3306/tcp | MySQL 8.0 数据库 |

### 2.2 数据库初始化

**首次安装**：访问 http://localhost:8080/install.php 通过安装向导完成数据库初始化。

安装向导会自动创建 82 张 ShopXO 原生表。

**安装后**：管理后台入口会被随机重命名为 `adminwlmqhs.php`。

### 2.3 后端配置

#### 环境变量配置

复制 `.env.example` 为 `.env`：

```powershell
cd shopxo-backend
copy .env.example .env
```

`.env` 文件内容说明：

> **值的来源说明**：下表中"默认值"列展示的是 `.env.example` 模板中的示例值。实际运行值由安装向导或你的 `.env` 决定。

## 五、权限配置步骤

修改 `sxo_power` 表后**必须**清除缓存，否则后台看不到新菜单。

```powershell
docker exec shopxo-php rm -rf /var/www/html/runtime/cache/*

```

方式二：后台界面清除

1. 登录管理后台：http://localhost:8080/adminwlmqhs.php
2. 进入：系统管理 → 缓存管理
3. 点击"清除所有缓存"

### 5.2 分配权限

1. 登录管理后台：http://localhost:8080/adminwlmqhs.php
2. 进入：系统管理 → 角色管理
3. 找到"管理员"角色，点击编辑
4. 在权限树中勾选"运营"及其所有子权限：
   - ✅ 运营
     - ✅ 活动管理（含所有三级权限）
     - ✅ 报名管理（含所有三级权限）
     - ✅ 邀请管理（含所有三级权限）
     - ✅ 数据报表
5. 保存
6. 按 `Ctrl+F5` 强制刷新浏览器

## 七、生产环境部署步骤

### 7.1 域名配置

1. 购买域名并在 DNS 中添加 A 记录指向服务器 IP
2. 修改 Nginx 配置 `docker/nginx/default.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为实际域名
    # ... 其余配置不变
}
```

### 7.2 HTTPS 配置

微信小程序正式环境**必须**使用 HTTPS。

1. 申请 SSL 证书（推荐 Let's Encrypt 免费证书或云服务商免费证书）
2. 修改 Nginx 配置添加 HTTPS：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    # SSL 优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    root /var/www/html/public;
    index index.php index.html;

    client_max_body_size 50m;

    location / {
        if (!-e $request_filename) {
            rewrite ^(.*)$ /index.php?s=/$1 last;
        }
    }

    location ~ \.php$ {
        fastcgi_pass php:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.ht { deny all; }
    location ~ /runtime/ { deny all; }
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}
```

3. 在 docker-compose.yml 中添加 SSL 证书卷挂载和 443 端口映射
4. 重启 Nginx 容器

### 7.3 生产环境安全加固

| 项目         | 操作                                           | 说明                         |
| ------------ | ---------------------------------------------- | ---------------------------- |
| 数据库密码   | 修改 docker-compose.yml 中 MYSQL_ROOT_PASSWORD | 同时修改 config/database.php |
| APP_DEBUG    | 设为 false                                     | 关闭调试模式                 |
| 管理员密码   | 修改 admin 默认密码                            | 后台 → 系统管理 → 管理员管理 |
| 管理后台入口 | 重命名 adminwlmqhs.php 为随机字符串            | 已在安装时自动重命名         |
| MySQL 端口   | 移除 3306 端口映射                             | 仅容器内访问                 |
| CORS         | 配置 Nginx CORS 头                             | H5 端需要                    |

### 7.4 前端生产环境配置

#### API 地址配置

编辑 `shopxo-uniapp/.env.production` 文件，替换占位符为实际域名：

```ini
UNI_APP_REQUEST_URL=https://api.yunxi.com/
VUE_APP_UNI_APP_REQUEST_URL=https://api.yunxi.com/

# 可选：静态资源独立域名
UNI_APP_STATIC_URL=https://static.yunxi.com/
VUE_APP_UNI_APP_STATIC_URL=https://static.yunxi.com/
```

**注意**：

- 必须使用 `https://`（微信小程序正式环境强制 HTTPS）
- 末尾 `/` 必须保留
- `.env.production` 文件已被 Git 忽略，不会提交到仓库

| MYSQL_COLLATION     | utf8mb4_unicode_ci        | 数据库默认排序规则（根目录 `.env.example` 中的值）            |

> **注意**：上述值为 `docker-compose.yml` 中的 fallback/default 值。如果你使用根目录 `.env` 文件，以 `.env` 中的值为准。

## 九、常见问题排查

> **提示**：本节命令中的 `-proot123456 shopxo` 为示例值，请替换为你实际的 MySQL 密码和数据库名。

### 9.1 Docker 启动失败

**症状**：`docker compose up -d` 后容器未启动或立即退出

**排查步骤**：

```powershell
# 查看所有容器状态（包括已停止的）
docker compose ps -a

# 查看容器日志
docker logs shopxo-nginx
docker logs shopxo-php
docker logs shopxo-mysql

# 常见原因：
# 1. 端口被占用 → 修改 docker-compose.yml 中的端口映射
# 2. Docker Desktop 未启动 → 启动 Docker Desktop
# 3. WSL2 未启用 → 以管理员身份运行：wsl --install
```

**端口冲突处理**：

```powershell
# 检查 8080 端口占用
netstat -ano | findstr :8080
# 检查 13306 端口占用（宿主机默认映射端口，3306 是容器内监听端口）
netstat -ano | findstr :13306
```

### 9.2 数据库连接失败

**症状**：后端页面报 500 错误或"数据库连接失败"

**排查步骤**：

```powershell
# 1. 检查 MySQL 容器是否就绪
docker exec shopxo-mysql mysql -uroot -proot123456 -e "SELECT 1;"

# 2. 检查数据库配置
docker exec shopxo-php cat /var/www/html/config/database.php

# 3. 确认 hostname 为 mysql（不是 localhost）
# Docker 网络内通过服务名访问，不是 127.0.0.1

# 4. 查看 PHP 错误日志
docker exec shopxo-php ls /var/www/html/runtime/log/
docker exec shopxo-php cat /var/www/html/runtime/log/最新日期.log
```

### 9.3 小程序白屏

**症状**：微信开发者工具中打开小程序后显示白屏

**排查步骤**：

1. 检查 `shopxo-uniapp/.env.development` 中 `UNI_APP_REQUEST_URL` 是否正确配置
2. 确认后端服务正常运行（浏览器访问 http://localhost:8080/ 或你的实际后端地址）
3. 微信开发者工具中确认"不校验合法域名"已勾选
4. 检查控制台是否有网络请求错误
5. 清除小程序缓存：工具栏 → 清缓存 → 全部清除

### 9.4 后台新菜单不显示

**症状**：执行迁移 SQL 后，后台左侧菜单看不到"运营"

**排查步骤**：

1. 清除缓存（见第五章 5.1 节）
2. 检查管理员角色是否已分配"运营"权限（见第五章 5.2 节）
3. 浏览器按 `Ctrl+F5` 强制刷新
4. 检查 `sxo_power` 表中是否有"运营"记录：

```powershell
docker exec shopxo-mysql mysql -uroot -proot123456 -e "SELECT id, name, pid, is_show FROM sxo_power WHERE name='运营';" shopxo
```

### 9.5 PHP 扩展缺失

**症状**：页面报错提示缺少 PHP 扩展

**排查步骤**：

```powershell
# 查看已安装扩展
docker exec shopxo-php php -m

# 修改 docker/php/Dockerfile 添加扩展后重新构建
docker compose up -d --build php
```

### 9.6 Nginx 502 Bad Gateway

**症状**：访问后端返回 502 错误

**排查步骤**：

```powershell
# 检查 PHP-FPM 容器是否运行
docker ps | findstr shopxo-php

# 查看 PHP-FPM 日志
docker logs shopxo-php

# 重启 PHP 容器
docker compose restart php
```

### 9.7 Composer 依赖问题

**症状**：后端报类找不到等错误

```powershell
# 重新安装依赖
docker exec shopxo-php composer install --no-dev

# 清除运行时缓存
docker exec shopxo-php rm -rf /var/www/html/runtime/*
```
