## 三、报名流程

### 3.1 用户端流程

```
用户浏览活动列表 → 选择活动 → 进入活动详情页 → 点击"立即报名" → 填写报名表单 → 提交报名
```

**详细步骤**：

1. **活动列表页**（`pages/activity/activity.vue`）：用户可按分类/阶段筛选活动
2. **活动详情页**（`pages/activity-detail/activity-detail.vue`）：展示活动完整信息，底部有"立即报名"按钮
3. **报名表单页**（`pages/activity-signup/activity-signup.vue`）：填写报名信息
4. **提交报名**：调用 `POST /api.php?s=activity/signup`

### 3.2 报名提交校验逻辑（ActivityService::ActivitySignup）

校验顺序：
1. 参数校验：activity_id、name（1-60字符）、phone（1-30字符）必填
2. 活动存在性校验：活动必须存在、启用、未删除
3. 报名时间校验：当前时间需在报名时间段内
4. 重复报名校验：同一用户同一活动不能重复报名（status in [0,1]）
5. 人数上限校验：max_count > 0 时，signup_count 不能超过 max_count
6. 插入报名记录：status=0（待确认），signup_count +1

### 3.3 取消报名（ActivityService::SignupCancel）

- 将报名记录 status 改为 2（已取消），is_delete_time 设为当前时间
- 活动 signup_count -1
- 仅允许本人取消，且只能取消 status in [0,1] 的记录

### 3.4 后台确认

当前报名提交后 status=0（待确认），后台管理员可在报名管理中查看并手动确认。确认操作通过状态更新实现。

## 四、签到核销流程

### 4.1 操作路径

```
后台管理 → 运营 → 报名管理 → 查看报名列表 → 点击签到按钮
```

### 4.2 签到逻辑（ActivityService::SignupCheckin）

1. 参数校验：id（报名记录ID）必填
2. 查询报名记录：checkin_status 必须为 0（未签到）
3. 更新记录：checkin_status → 1，checkin_time → 当前时间，upd_time → 当前时间
4. 返回签到成功

**幂等性**：已签到的记录（checkin_status=1）再次签到会返回"报名记录不存在或已签到"，不会重复签到。

### 4.3 后台管理入口

- 控制器：`app/admin/controller/Activitysignup.php`
- 签到接口：`POST /adminwlmqhs.php?s=activitysignup/checkin`，参数 `id`
- 签到按钮在报名管理列表的操作列中

## 五、导出功能说明

### 5.1 导出接口

- 路径：`GET/POST /adminwlmqhs.php?s=activitysignup/export`
- 控制器：`Activitysignup::Export()`

### 5.2 筛选条件

| 参数 | 类型 | 说明 |
|------|------|------|
| activity_id | int | 按活动ID筛选 |
| status | int | 按报名状态筛选（0/1/2） |
| checkin_status | int | 按签到状态筛选（0/1） |

### 5.3 导出格式

CSV文件，UTF-8 BOM编码，文件名格式 `activity_signup_YYYYMMDDHHmmss.csv`

**导出列**：

| 列名 | 数据来源 |
|------|----------|
| 报名ID | signup.id |
| 活动标题 | activity.title |
| 姓名 | signup.name |
| 手机 | signup.phone |
| 阶段 | stage 中文映射 |
| 预产期 | due_date 格式化为 Y-m-d |
| 宝宝月龄 | baby_month_age + "个月" |
| 报名状态 | status 中文映射（待确认/已确认/已取消） |
| 签到状态 | checkin_status 中文映射（未签到/已签到） |
| 报名时间 | add_time 格式化 |
| 签到时间 | checkin_time 格式化 |

## 六、前端页面说明

### 6.1 活动详情页（activity-detail.vue）

**文件**：`shopxo-uniapp/pages/activity-detail/activity-detail.vue`

**功能**：
- 顶部自定义导航栏（滚动变色效果）
- 封面图 + 阶段标签
- 基础信息卡片：标题、时间、地址、报名人数/上限、报名截止时间、价格、联系人
- 活动详情富文本（mp-html 渲染）
- 适合人群说明
- 用户晒单横向滚动
- 互动区（点赞/评论/分享）
- 底部操作栏：收藏、海报、立即报名按钮

**报名按钮状态**：
- 名额未满：显示"立即报名"，粉色渐变背景
- 名额已满：显示"名额已满"，灰色背景，disabled

**跳转**：点击报名 → `/pages/activity-signup/activity-signup?id={activity_id}`

### 6.2 活动报名页（activity-signup.vue）

**文件**：`shopxo-uniapp/pages/activity-signup/activity-signup.vue`

**功能**：
- 活动摘要卡片（标题、时间、价格）
- 报名表单：
  - 姓名（必填，text，max 20）
  - 手机号（必填，number，max 11，正则校验 `/^1[3-9]\d{9}$/`）
  - 当前阶段（必填，picker：备孕/孕期/产后）
  - 预产期（孕期时必填，date picker，起始日期为当天）
  - 宝宝月龄（产后时必填，picker：1-36个月）
  - 备注（选填，textarea，max 200）

**动态字段逻辑**：
- 选择"备孕"：不显示额外字段
- 选择"孕期"：显示预产期选择器
- 选择"产后"：显示宝宝月龄选择器（1-36个月）

**提交逻辑**：
1. 前端校验通过后调用 `POST /api.php?s=activity/signup`
2. 成功：Toast "报名成功"，1.5秒后返回上一页
3. 失败：Toast 错误信息

## 九、动态字段逻辑

### 9.1 前端表单动态显隐

在 `activity-signup.vue` 中，根据用户选择的"当前阶段"动态显示不同字段：

| 选中阶段 | 显示字段 | 必填 | 字段类型 |
|----------|----------|------|----------|
| prepare（备孕） | 无额外字段 | - | - |
| pregnancy（孕期） | due_date（预产期） | 是 | date picker，起始日期为当天 |
| postpartum（产后） | baby_month_age（宝宝月龄） | 是 | picker，1-36个月 |

### 9.2 后端存储

- `due_date`：存储为 int unsigned 时间戳，前端传日期字符串（如 `2026-08-15`），后端 `ActivityService::ActivitySignup` 中暂未做日期→时间戳转换，需在后续对接时补充
- `baby_month_age`：存储为 int unsigned 月数，前端传如 `6`，后端直接存储

### 9.3 导出时的格式化

- `due_date`：格式化为 `Y-m-d`
- `baby_month_age`：拼接为 `X个月`
