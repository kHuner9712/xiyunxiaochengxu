# 63-活动详情页接口契约说明

## 改了哪些文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `shopxo-backend/app/api/controller/Activity.php` | 修改 | Detail 接口返回结构统一，增加 is_favored/is_liked/like_count/comment_count/user_shares 字段 |
| `shopxo-uniapp/pages/activity-detail/activity-detail.vue` | 修改 | 移除 mock 数据，对齐接口结构，未实现功能标注"暂未开放" |
| `shopxo-uniapp/pages/activity-signup/activity-signup.vue` | 修改 | 移除 mock 活动数据，从接口获取活动摘要 |
| `shopxo-backend/app/service/ActivityService.php` | 修改 | ActivityDetail 增加 is_enable/is_delete_time 过滤 |

## 为什么改

1. **前后端接口不对齐**：前端读取 `data.activity` / `data.user_shares` / `is_favored` / `is_liked`，后端返回 `data.data`，结构不匹配
2. **mock 数据残留**：activity-detail.vue 的 data() 中硬编码了完整的活动演示数据，接口失败时回退到 mock 数据，误导测试
3. **未实现功能伪装**：收藏、点赞、评论、海报功能在前端有交互但后端未实现，用户操作后无真实效果
4. **逻辑删除活动可读取**：已删除/已禁用的活动通过详情接口仍可读取

### 字段说明

| 字段 | 类型 | 说明 | 当前状态 |
|------|------|------|----------|
| activity | object | 活动详情对象 | ✅ 已实现 |
| is_favored | boolean | 当前用户是否收藏 | ⚠️ 部分实现（查询 GoodsFavor 表，type=activity） |
| is_liked | boolean | 当前用户是否点赞 | ❌ 未实现，固定返回 false |
| like_count | int | 点赞数 | ❌ 未实现，固定返回 0 |
| comment_count | int | 评论数 | ❌ 未实现，固定返回 0 |
| user_shares | array | 用户晒单列表 | ❌ 未实现，固定返回空数组 |

### activity 对象中的前端映射

| 后端字段 | 前端使用 | 映射逻辑 |
|----------|----------|----------|
| title | activity.title | 直接映射 |
| cover | activity.cover | 直接映射 |
| stage | activity.stage_name / activity.stage_class | 需前端做映射（prepare→备孕/muying-stage-tag--prepare 等） |
| start_time_text + end_time_text | activity.time | 前端拼接为 "开始时间 ~ 结束时间" |
| address | activity.address | 直接映射 |
| signup_count / max_count | activity.signup_count / activity.max_count | 直接映射 |
| signup_end_time_text | activity.signup_deadline | 直接映射 |
| price | activity.price | 直接映射，0 显示"免费" |
| contact_name | activity.organizer | 字段名不同，需前端映射 |
| contact_phone | activity.organizer_phone | 字段名不同，需前端映射 |
| suitable_crowd | activity.suitable_crowd | 直接映射 |
| content | activity.content | 直接映射，mp-html 渲染 |

## 迁移步骤

1. 直接部署代码即可，无需数据库结构变更
2. 部署后活动详情页将从真实接口获取数据

## 风险点

1. **字段名映射**：后端返回 `contact_name`/`contact_phone`，前端使用 `organizer`/`organizer_phone`，当前 activity-detail.vue 直接读取 activity 对象的字段，需确保字段名一致或前端做映射
2. **收藏功能**：is_favored 查询使用了 GoodsFavor 表的 type='activity'，需确认该表是否支持此类型。如果不支持，is_favored 将始终为 false
3. **空数据处理**：移除 mock 数据后，如果接口返回失败，页面将显示空白。这是预期行为——不应伪装数据存在
4. **activity-signup.vue**：新增的 get_activity_summary 方法依赖 detail 接口返回 start_time_text/end_time_text，需确认 ActivityList 输出包含这些字段

## 如何验证

1. 打开活动详情页，确认内容来自真实接口（检查网络请求）
2. 检查接口返回结构与前端读取逻辑一致
3. 确认页面中不再保留 mock 默认主数据
4. 点击收藏/点赞/海报按钮，确认显示"暂未开放"提示
5. 访问已删除活动的详情页，确认返回"活动不存在"
6. 打开活动报名页，确认活动标题/时间/价格来自接口
