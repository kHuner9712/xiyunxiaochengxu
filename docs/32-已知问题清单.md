## P0 紧急 — 阻塞上线

### P0-01: App.vue request_url 初始为空字符串

| 项目 | 内容 |
|------|------|
| 编号 | P0-01 |
| 描述 | `shopxo-uniapp/App.vue` 中 `request_url` 初始值为空字符串 `''`，首次启动时如果后端配置接口未返回该值，所有 API 请求将失败，导致小程序白屏或功能不可用 |
| 影响范围 | 全部前端功能，所有 API 调用 |
| 当前状态 | 已手动修改为 `http://localhost:8080/`，但生产环境部署时仍需手动修改 |
| 修复建议 | 1. 在后端新增"系统初始化配置"接口，返回 request_url、static_url 等基础配置；2. App.vue onLaunch 中优先从接口获取，fallback 到本地配置；3. 或改为从环境变量/编译配置注入 |

### P0-02: 数据库密码硬编码在 config/database.php

| 项目 | 内容 |
|------|------|
| 编号 | P0-02 |
| 描述 | `shopxo-backend/config/database.php` 中数据库密码 `root123456` 明文硬编码，且该文件被 git 跟踪（安装向导生成后未加入 .gitignore），存在密码泄露风险 |
| 影响范围 | 数据库安全，生产环境部署 |
| 当前状态 | 密码明文存在于代码仓库中 |
| 修复建议 | 1. 将 database.php 加入 .gitignore；2. 改用环境变量读取密码（`env('DB_PASSWORD', 'root123456')`）；3. 生产环境使用强密码；4. 审查 git 历史中是否已泄露 |

### P1-05: 邀请码反查全表扫描性能问题

| 编号 | P1-05 |

## P2 中 — 质量风险

### P2-01: ActivitySignup/SignupCancel 缺少数据库事务

| 项目 | 内容 |
|------|------|
| 编号 | P2-01 |
| 描述 | `ActivityService::ActivitySignup()` 中报名记录插入和 `signup_count +1` 不是原子操作，`SignupCancel()` 中取消和 `signup_count -1` 也不是原子操作，高并发下可能导致计数不一致 |
| 影响范围 | 报名人数统计准确性 |
| 当前状态 | 无事务保护 |
| 修复建议 | 1. 使用 `Db::startTrans() / Db::commit() / Db::rollback()` 包裹操作；2. 或使用 `Db::name('Activity')->where('id', $id)->inc('signup_count')->update()` 配合行锁 |

### P2-02: Docker MySQL 端口暴露

| 项目 | 内容 |
|------|------|
| 编号 | P2-02 |
| 描述 | `docker-compose.yml` 中 MySQL 3306 端口映射到宿主机，生产环境下存在安全风险，外部可直接访问数据库 |
| 影响范围 | 数据库安全 |
| 当前状态 | 端口暴露 |
| 修复建议 | 1. 生产环境移除 `ports: - "3306:3306"` 配置；2. 仅通过 Docker 内部网络访问；3. 如需外部调试，使用 `127.0.0.1:3306:3306` 限制本机访问 |

### P2-03: 缺少 Redis 服务

| 项目 | 内容 |
|------|------|
| 编号 | P2-03 |
| 描述 | Docker 环境中未配置 Redis 服务，PHP 已安装 redis 扩展但无法使用，ShopXO 缓存驱动回退到文件缓存，性能较差 |
| 影响范围 | 系统缓存性能，高并发场景 |
| 当前状态 | 使用文件缓存 |
| 修复建议 | 1. docker-compose.yml 新增 Redis 服务；2. 修改 ShopXO 缓存配置为 Redis 驱动；3. Session 也可改用 Redis 存储 |

### P2-04: CORS 未配置

| 项目 | 内容 |
|------|------|
| 编号 | P2-04 |
| 描述 | Nginx 和后端均未配置 CORS 响应头，H5 端跨域请求会被浏览器拦截 |
| 影响范围 | H5 端功能 |
| 当前状态 | H5 端无法正常调用 API |
| 修复建议 | 1. Nginx 配置中添加 CORS 头：`add_header Access-Control-Allow-Origin *; add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS'; add_header Access-Control-Allow-Headers '*';`；2. 处理 OPTIONS 预检请求 |

### P2-05: Devtest 控制器包含调试方法

| 项目 | 内容 |
|------|------|
| 编号 | P2-05 |
| 描述 | `app/api/controller/Devtest.php` 包含调试和测试方法，生产环境不应暴露 |
| 影响范围 | API 安全 |
| 当前状态 | 控制器存在于代码中 |
| 修复建议 | 1. 生产环境删除该文件；2. 或在路由配置中禁用该控制器；3. 通过环境变量控制是否加载 |
