# 真机测试清单

> 本清单面向非技术用户，按顺序执行即可完成核心功能验证。
> 每一步请记录：✅通过 / ❌失败 / ⚠️异常

---

## 一、环境准备

- [ ] Docker Desktop 已启动
- [ ] 打开终端执行 `docker ps`，确认 3 个容器运行中（shopxo-nginx, shopxo-php, shopxo-mysql）
- [ ] 浏览器访问 http://localhost:8080/ 能打开页面
- [ ] 微信开发者工具已安装

---

## 二、后台操作测试

### 2.1 登录后台

- [ ] 打开浏览器，访问 http://localhost:8080/adminwlmqhs.php
- [ ] 输入账号 `admin`，密码 `admin123`
- [ ] 点击登录
- [ ] **预期**：成功进入后台首页，左侧显示菜单

### 2.2 查看报名管理

- [ ] 左侧菜单点击「运营」→「报名管理」
- [ ] **预期**：显示报名记录列表，包含姓名、手机、阶段、签到状态等列

### 2.3 执行签到

- [ ] 找到一条签到状态为「未签到」的记录
- [ ] 点击操作栏的「签到」按钮
- [ ] **预期**：提示签到成功，该记录签到状态变为「已签到」，签到时间更新

### 2.4 导出数据

- [ ] 在报名管理页面点击「导出」按钮
- [ ] **预期**：浏览器下载 CSV 文件，用 Excel 打开可正常显示中文

## 三、小程序端测试

### 3.1 打开小程序

- [ ] 打开微信开发者工具
- [ ] 导入项目，选择 `shopxo-uniapp` 目录
- [ ] 右上角「详情」→「本地设置」→ 勾选「不校验合法域名」
- [ ] 编译运行
- [ ] **预期**：小程序启动，底部显示5个Tab（首页/商城/活动/购物车/我的）

### 3.2 首页浏览

- [ ] 查看首页内容
- [ ] **预期**：显示 Banner、阶段导航（备孕/孕期/产后）、推荐活动、推荐商品等模块

### 3.3 登录

- [ ] 点击底部「我的」Tab
- [ ] 点击登录按钮
- [ ] 输入账号 `testuser01`，密码 `123456`
- [ ] **预期**：登录成功，显示用户昵称和头像

### 3.4 活动列表

- [ ] 点击底部「活动」Tab
- [ ] **预期**：显示活动列表，每个活动有标题、时间、地点、报名按钮

### 3.5 活动详情

- [ ] 点击任一活动进入详情
- [ ] **预期**：显示封面图、标题、时间、地点、活动介绍、适合人群、报名按钮

### 3.6 提交报名

- [ ] 在活动详情页点击「立即报名」
- [ ] 填写姓名、手机号、选择阶段
- [ ] 点击提交
- [ ] **预期**：提示报名成功

### 3.7 我的报名

- [ ] 进入「我的」页面
- [ ] 点击「我的报名」或「我的活动」
- [ ] **预期**：显示刚才报名的活动记录
