# 孕禧小程序 — 版本发布清单

> 从"代码完成"到"正式上线"的完整操作清单，每一步都必须确认。

---

## 一、提交审核前（代码冻结后）

### 1.1 代码与配置确认

| # | 操作 | 确认方式 | 负责人 |
|---|------|---------|--------|
| 1 | 确认所有代码已推送到 main 分支 | `git log --oneline -5` | 开发 |
| 2 | 确认 `manifest.json` 中 `appid` 已填入正式 AppID | 检查文件 | 开发 |
| 3 | 确认 `UNI_APP_REQUEST_URL` 指向正式 API 域名 | 检查环境变量 | 开发 |
| 4 | 确认后端 `APP_DEBUG = false` | 检查 `.env` | 运维 |
| 5 | 确认后端 `show_error_msg = false` | 检查 `config/app.php` | 运维 |
| 6 | 确认 `install.php` 已删除 | `ls public/install.php` → 不存在 | 运维 |
| 7 | 确认数据库 migration 已全部执行 | 运行 `check-server.sh` 全部 PASS | 运维 |
| 8 | 确认初始化 SQL 已执行 | 活动/反馈/配置项有数据 | 运维 |

### 1.2 内容确认

| # | 操作 | 确认方式 | 负责人 |
|---|------|---------|--------|
| 1 | 首页轮播图 ≥3 张 | 打开首页检查 | 运营 |
| 2 | 首页导航 ≥4 个 | 打开首页检查 | 运营 |
| 3 | 活动数据 ≥4 条 | 活动列表页检查 | 运营 |
| 4 | 商品分类含阶段关键词 | 切换阶段标签有不同商品 | 运营 |
| 5 | 知识文章 ≥3 篇 | 首页检查 | 运营 |
| 6 | 妈妈说 ≥3 条 | 首页检查 | 运营 |
| 7 | 邀请奖励积分非0 | 邀请页检查 | 运营 |
| 8 | 客服电话为真实号码 | 活动详情检查 | 运营 |
| 9 | 隐私政策可打开 | 登录页点击检查 | 运营 |
| 10 | 注册协议可打开 | 登录页点击检查 | 运营 |
| 11 | Logo 为孕禧品牌 | 全局检查 | 运营 |

### 1.3 构建与上传

| # | 操作 | 说明 |
|---|------|------|
| 1 | HBuilderX 发行 → 小程序-微信 | 设置正式环境变量 |
| 2 | 微信开发者工具打开构建产物 | 确认无编译错误 |
| 3 | 本地调试确认接口可通 | Network 面板检查 |
| 4 | 真机预览确认 | 扫码在手机上检查 |
| 5 | 点击"上传" | 版本号 `1.0.0`，备注"首次发布" |

## 三、正式发布前（发布按钮点击前最后确认）

| # | 操作 | 说明 |
|---|------|------|
| 1 | 确认审核版本与体验版一致 | 避免审核后改代码导致版本不一致 |
| 2 | 确认后端服务稳定 | 检查后端日志无异常 |
| 3 | 确认数据库连接正常 | `check-server.sh` 快速检查 |
| 4 | 确认 CDN/静态资源可访问 | 图片/Logo 正常加载 |
| 5 | 通知团队发布时间 | 确保有人值守 |

## 四、发布后 1 小时内重点观察

| # | 观察项 | 检查方式 | 异常处理 |
|---|--------|---------|---------|
| 1 | 小程序能否正常打开 | 真机扫码进入 | 检查后端服务/域名配置 |
| 2 | 登录是否正常 | 点击微信登录 | 检查 AppID/AppSecret 配置 |
| 3 | 首页内容是否正常 | 检查轮播/活动/商品 | 检查后台数据/接口 |
| 4 | 活动报名是否正常 | 提交一次报名 | 检查后端日志 |
| 5 | 邀请页是否正常 | 查看邀请码和奖励值 | 检查配置项 |
| 6 | 后端错误日志 | 查看 runtime/log/ | 有异常立即排查 |
| 7 | 数据库连接数 | `SHOW PROCESSLIST` | 异常高则检查慢查询 |
| 8 | 用户反馈 | 客服渠道 | 有问题立即响应 |

### 紧急回滚方案

如果发布后出现严重问题：

1. **前端回滚**：微信后台 → 版本管理 → 回退到上一版本（审核版）
2. **后端回滚**：`git revert HEAD` → 重新部署 → 重启 PHP-FPM
3. **数据库回滚**：从备份恢复 `mysql -u root -p shopxo < backup_XXXXXXXXXX.sql`
