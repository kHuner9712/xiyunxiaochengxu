# 后台首次登录配置清单

> 适用阶段：后端部署后首次配置  
> 执行人：运营/产品（非开发人员）  
> 输入物：后台管理员账号密码  
> 输出物：配置完整的后台（可展示内容的首页）  
> 最后更新：2026-04-25

---

## 前提

- 后端已部署，后台可访问（如 `http://你的域名/admin.php`）
- 已有管理员账号密码
- 已执行 `docs/muying-admin-power-migration.sql`

---

## 第 1 步：修改默认密码

| 项目 | 内容 |
|------|------|
| 位置 | 登录后 → 右上角头像 → 修改密码 |
| 操作 | 输入旧密码 → 输入新密码 → 确认 |
| 验证 | 退出重新登录，用新密码成功登录 |

---

## 第 2 步：确认菜单完整

| 项目 | 内容 |
|------|------|
| 位置 | 左侧菜单栏 |
| 操作 | 确认出现"孕禧运营"分组，包含以下 6 个子菜单 |
| 验证 | 逐个点击，确认页面正常加载 |

**必须存在的菜单项**：
1. 活动管理
2. 活动报名
3. 邀请奖励
4. 邀请配置
5. 用户反馈
6. 功能开关
7. 孕禧数据看板

**如果菜单不完整**：联系开发人员确认 `muying-admin-power-migration.sql` 是否已执行。

---

## 第 3 步：确认 MUYING_PRIVACY_KEY 已配置

| 项目 | 内容 |
|------|------|
| 位置 | `shopxo-backend/.env` → `MUYING_PRIVACY_KEY` |
| 操作 | 确认已填入 64 位 hex 密钥（可用 `php -r "echo bin2hex(openssl_random_pseudo_bytes(32));"` 生成） |
| 验证 | 后台活动报名 → 提交报名 → 不报"隐私配置异常" |

> **重要**：MUYING_PRIVACY_KEY 为必填项。缺失或过短时，活动报名、反馈提交等涉及敏感数据的写入操作将被阻断，返回"系统隐私配置异常"错误。

---

## 第 4 步：配置站点名称

| 项目 | 内容 |
|------|------|
| 位置 | 系统设置 → 站点设置 → 基本设置 |
| 操作 | 填写站点名称（如"孕禧"） |
| 保存 | 点击"保存" |

---

## 第 4 步：配置客服电话

| 项目 | 内容 |
|------|------|
| 位置 | 系统设置 → 手机端配置 → 基础配置 |
| 操作 | 找到"客服电话"，填入真实客服电话 |
| 保存 | 点击"保存" |
| 验证 | 前端"我的"页面 → "联系客服"入口可点击并拨号 |

---

## 第 5 步：配置隐私弹窗文案

| 项目 | 内容 |
|------|------|
| 位置 | 系统设置 → 小程序配置 → 微信小程序 |
| 操作 | 找到"隐私弹窗文案"，填入以下内容 |
| 保存 | 点击"保存" |

**推荐文案**：
```
为了向您提供活动签到、收货地址选择等服务，我们需要获取您的位置信息；为了更换头像或上传反馈图片，我们需要访问您的相册或摄像头。您可以拒绝授权，不影响其他功能使用。
```

---

## 第 6 步：配置功能开关

| 项目 | 内容 |
|------|------|
| 位置 | 孕禧运营 → 功能开关 |
| 操作 | 确认以下开关为"开启"状态 |
| 保存 | 点击"保存" |

**一期核心功能（必须开启）**：
- ☑ 活动报名
- ☑ 邀请裂变
- ☑ 官方内容
- ☑ 用户反馈

**一期不开放功能（保持关闭）**：
- ☐ 多商户/多门店/分销/钱包/积分商城/UGC/会员付费
- ☐ 秒杀/优惠券/签到/积分兑换/礼品卡/送礼
- ☐ 视频/直播/智能工具
- ☐ 互联网医院/发票/实名认证/扫码支付/投诉

---

## 第 7 步：配置邀请奖励

| 项目 | 内容 |
|------|------|
| 位置 | 孕禧运营 → 邀请配置 |
| 操作 | 填写以下配置项 |
| 保存 | 点击"保存" |

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 首单奖励积分 | 50-100 | 邀请人获得的积分奖励 |
| 自动发放 | ☑ 开启 | 首单完成后自动发放奖励 |
| 每日上限 | 10 | 每人每日最多发放奖励次数 |
| 邀请口号 | 邀请好友，共享好孕 | 邀请页标题文案 |

---

## 第 8 步：配置协议内容

| 项目 | 内容 |
|------|------|
| 位置 | 系统设置 → 协议管理 |
| 操作 | 逐个检查并填写 |

### 8.1 隐私政策
- 点击"隐私政策"标签
- 填写完整的隐私政策内容（必须包含：位置信息、相册/摄像头、手机号的使用说明）
- 保存

### 8.2 用户协议
- 点击"注册协议"标签
- 填写完整的用户协议内容
- 保存

**验证**：前端登录页 → 点击"隐私政策"/"用户协议"链接 → 确认内容正常显示

---

## 第 9 步：录入首批内容

| 项目 | 内容 |
|------|------|
| 参考 | `docs/release/seed-content-template.md` |

### 9.1 创建活动
- 孕禧运营 → 活动管理 → 添加活动
- 按模板填写，至少 1 个

### 9.2 创建商品
- 商品管理 → 添加商品
- 按模板填写，至少 2 个

### 9.3 创建文章
- 内容管理 → 文章管理 → 添加文章
- 按模板填写，至少 1 篇

---

## 第 11 步：最终验证

| 验证项 | 方法 |
|--------|------|
| 后台菜单完整 | 左侧菜单 7 个子菜单全部可点击 |
| 站点名称 | 前端标题栏显示"孕禧" |
| 客服电话 | 前端"我的" → "联系客服"可拨号 |
| 隐私弹窗 | 首次打开小程序弹出隐私授权 |

| 邀请配置 | 前端邀请页显示邀请码和奖励说明 |
| 协议内容 | 登录页协议链接可点击查看 |
| 首页内容 | 活动商品文章妈妈说区块有内容 |
| 数据看板 | 孕禧运营 → 孕禧数据看板 → 页面正常加载，6 大分区指标均显示（无数据时显示 0） |
| 数据看板时间筛选 | 切换今日/昨日/近7天/近30天/自定义，指标数值刷新 |

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 左侧菜单没有"孕禧运营" | SQL 未执行 | 执行 `docs/muying-admin-power-migration.sql` |
| 点击菜单报 404 | 控制器文件缺失 | 检查 `app/admin/controller/` 下对应 PHP 文件 |
| 功能开关页面空白 | 视图文件缺失 | 检查 `app/admin/view/default/featureswitch/index.html` |
| 保存配置后前端无变化 | 缓存未清除 | 后台 → 系统设置 → 缓存管理 → 清除缓存 |
| 客服电话保存了但前端不显示 | 配置 key 错误 | 确认 `common_app_customer_service_tel` 配置项存在 |
