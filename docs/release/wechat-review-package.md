## 2. 核心功能简介（提审说明文案）

**直接复制以下内容到微信后台"功能页面"说明框：**

```
孕禧小程序是一款面向备孕、孕期、产后妈妈的母婴服务平台。

核心功能：
1. 首页推荐：根据用户当前孕育阶段（备孕/孕期/产后）智能推荐商品、活动、知识文章

- 邀请有礼：用户中心 → 邀请有礼 → 查看邀请码和奖励规则
- 商品浏览：首页切换"备孕/孕期/产后"标签 → 查看阶段推荐商品
- 个人中心：底部Tab"我的" → 查看阶段标签、订单、积分等

本小程序无需测试账号即可体验全部功能。
登录使用微信授权+手机号绑定方式，手机号仅用于身份验证和活动通知，不会用于其他用途。

## 4. 邀请有礼、活动报名、用户阶段采集的审核表述

### 4.1 邀请有礼

**提审表述**：

```
邀请有礼功能说明：
- 用户通过分享小程序给好友，好友注册成功后邀请人获得积分奖励
- 积分奖励仅用于平台内兑换商品或服务，不涉及现金提现
- 邀请关系基于邀请码建立，邀请码由系统自动生成
- 本功能不涉及多级分销，仅限一级邀请奖励
- 奖励积分值在页面明确展示，无隐藏规则
```

**关键合规点**：
- ❌ 不使用"裂变""拉新""返利"等敏感词
- ✅ 强调"一级邀请""积分奖励""无现金提现"
- ✅ 奖励规则透明展示

### 4.2 活动报名

**提审表述**：

```
活动报名功能说明：
- 用户可浏览平台组织的线下/线上母婴活动
- 报名时需填写姓名、手机号、孕育阶段等信息，用于活动确认和通知
- 报名前需同意隐私告知，明确信息收集目的和范围
- 活动信息由平台运营方发布，非用户生成内容
```

**关键合规点**：
- ✅ 报名页有独立隐私告知区块
- ✅ 收集信息有明确用途说明
- ✅ 活动非UGC，由平台发布

### 4.3 用户阶段采集

**提审表述**：

```
用户阶段信息说明：

## 5. 隐私与权限说明

### 5.1 当前使用的微信能力及用途

| 微信能力 | 用途 | 是否强制 | 用户拒绝后 |
|----------|------|:---:|-----------|
| `wx.login` | 小程序登录获取 openid | 是 | 无法登录，但用户仍可浏览首页公开内容 |
| `getPhoneNumber` | 获取手机号绑定账户 | 是（配置控制） | 可跳过绑定，但无法下单/报名 |
| `chooseAvatar` | 选择头像（个人资料页） | 否 | 使用默认头像，不影响任何功能 |
| `chooseImage` | 选择图片（修改头像/地址） | 否 | 不影响核心功能 |
| `getLocation` | 获取位置（门店自提） | 否 | 不影响核心功能，可手动选择地址 |
| `chooseLocation` | 选择位置（门店自提） | 否 | 同上 |
| `scanCode` | 扫码（核销等） | 否 | 不影响核心功能 |
| `openCustomerServiceChat` | 客服对话 | 否 | 不影响核心功能 |

### 5.2 requiredPrivateInfos 声明

当前 `manifest.json` 已声明：

```json
"requiredPrivateInfos": [
    "chooseLocation",
    "getLocation",
    "chooseAddress",
    "onLocationChange",
    "startLocationUpdate",
    "startLocationUpdateBackground"
]
```

**⚠️ 需补充声明**：如果提审时使用了 `chooseImage`/`chooseMedia`/`chooseAvatar`/`saveImageToPhotosAlbum`，需在微信后台"用户隐私保护指引"中补充对应接口。

### 5.3 登录链路审核备注

**当前登录流程**（改造后）：

```
用户点击"微信登录" → uni.login() 获取 code → 后端 code2session 获取 openid → 创建/登录用户（默认昵称）→ 检查是否需要绑定手机号 → 绑定手机号 → 进入系统
```

**提审备注**（直接复制到审核说明）：

```
登录说明：
1. 本小程序采用微信快捷登录方式，用户点击"微信登录"后系统通过 wx.login 获取 openid 完成身份识别
2. 用户无需授权头像昵称即可完成登录，系统自动分配默认昵称
3. 头像和昵称可在"个人中心-个人资料"页面自愿补充，不强制要求
4. 手机号绑定用于订单联系和活动通知，用户可在登录后绑定
5. 用户拒绝授权手机号不影响浏览首页、查看活动、阅读知识文章等核心功能
6. 本小程序不使用已废弃的 wx.getUserProfile 接口
```

## 8. 常见审核被拒原因与预防

| 被拒原因 | 本项目风险点 | 预防措施 |
|----------|-------------|---------|
| 隐私政策不完整 | 隐私政策 URL 未配置或内容不完整 | 确保后台 `agreement_userprivacy_url` 已配置且内容完整 |
| 诱导分享 | 邀请有礼可能被误判 | 邀请页文案避免"分享得""分享领"，使用"邀请好友注册" |
| 虚拟支付 | 积分兑换可能被误判 | 积分仅用于兑换实物商品，不涉及虚拟商品 |
| 登录强制授权 | 旧版 getUserProfile 强制授权 | 已改为默认昵称登录，头像昵称非强制 |
| 内容空白 | 首页/活动页无数据 | 确保初始化 SQL 已执行，后台已配置内容 |
| 测试数据 | 示例手机号 400-000-0000 | 替换为真实客服电话 |
| 类目不符 | 活动报名可能需"医疗"类目 | 确保服务类目包含"健康咨询"或"母婴护理" |
