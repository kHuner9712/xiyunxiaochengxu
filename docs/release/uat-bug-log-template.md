---

## 严重级别定义

| 级别 | 含义 | 示例 | 是否阻断上线 |
|------|------|------|:---:|
| P0 | 核心功能不可用，数据错误，安全漏洞 | 无法登录、报名失败、手机号泄露 | ✅ |
| P1 | 重要功能异常但有替代路径，体验严重受损 | 阶段标签不显示、邀请码复制失败 | 视情况 |
| P2 | 轻微体验问题，不影响核心功能 | 样式偏移、文案错误、动画卡顿 | ❌ |

## 状态流转

```
☐待修 → ☐已修 → ☐验证通过 → 关闭
                ↘ ☐验证失败 → 回到 ☐待修
```

- **待修**：问题已确认，等待开发修复
- **已修**：开发已提交修复代码
- **验证通过**：测试确认修复有效，关闭
- **验证失败**：修复无效或引入新问题，回到待修

---

## P0 Bug 上线阻断规则

1. 任何 P0 Bug 未修复 → **不可上线**
2. P0 修复后必须回归验证通过 → 才可继续发布流程
3. P0 修复引入新 P0 → **回退代码**，重新评估
4. P1 Bug 累计 ≥ 5 个 → 建议延期上线
