# 禧孕母婴用品私域商城小程序 - 产品需求文档

## 1. 文档概述

### 1.1 文档目的

本文档是"禧孕母婴用品私域商城小程序"的产品需求文档（PRD），旨在全面、清晰地描述产品的功能需求、非功能需求及数据需求，为产品设计、技术开发、测试验收提供统一的需求基准和参考依据。

### 1.2 适用范围

本文档适用于禧孕母婴用品私域商城小程序的全量功能模块，包括：

- 微信小程序用户端（面向C端消费者）
- 管理后台Web端（面向内部运营管理人员）

本文档不涉及第三方供应商端或分销商端，本平台为甲方自营商城，所有商品由禧孕文化传媒有限公司统一采购、定价、发货和售后。

## 2. 产品定位

### 2.1 产品名称

**禧孕母婴用品私域商城小程序**（以下简称"禧孕商城"）

### 2.2 运营主体

**禧孕文化传媒有限公司**

### 2.3 产品定位

禧孕商城是一个面向母婴人群的**私域购物平台**，依托微信小程序生态，为0-3岁宝宝家庭提供高品质、高性价比的母婴用品购物服务。平台采用**甲方自营模式**，所有商品由禧孕文化传媒有限公司统一采购、统一定价、统一仓储、统一发货、统一售后，确保商品品质和服务体验。

### 2.4 核心特点

| 特点 | 说明 |
|------|------|
| **自营模式** | 非多商户平台，所有商品由甲方统一管理，不存在第三方商家入驻 |
| **私域运营** | 依托微信生态，通过社群、公众号、朋友圈等私域渠道获客和运营 |
| **月龄推荐** | 基于宝宝月龄智能推荐适龄商品，解决新手父母"不知道买什么"的痛点 |
| **会员体系** | 完善的会员等级与积分体系，提升用户粘性和复购率 |
| **周期购提醒** | 针对高频复购商品（纸尿裤、奶粉等）提供周期提醒服务 |
| **活动丰富** | 限时折扣、满减满赠、新人礼包、组合套餐等多种营销玩法 |

### 2.5 目标用户

| 用户群体 | 特征描述 |
|---------|---------|
| 孕妈 | 孕期女性，关注待产包、孕期营养、产后恢复等 |
| 新手妈妈 | 0-6个月宝宝妈妈，需要专业指导和高品质用品 |
| 经验妈妈 | 6个月以上宝宝妈妈，注重性价比和复购便利 |
| 宝宝其他照护者 | 爸爸、爷爷奶奶、外公外婆等，代为购买 |

### 2.6 业务边界

- **包含**：商品展示与销售、订单与支付、物流与售后、会员与积分、营销活动、内容运营
- **不包含**：第三方商家入驻、社区交流/UGC、直播带货、跨境海淘、线下门店O2O

## 3. 小程序用户端功能需求

### 3.1 微信登录

#### 3.1.1 功能描述

用户通过微信授权快速登录小程序，首次登录需绑定手机号，系统自动获取微信头像和昵称作为默认用户信息。

#### 3.1.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 微信授权登录 | 点击登录按钮，调用微信 `wx.login` 获取 code，后端换取 openid 和 session_key；调用 `wx.getUserProfile` 获取用户微信头像和昵称 |
| 手机号绑定 | 首次登录弹出手机号授权弹窗，用户点击"允许"后通过微信手机号快速验证组件获取手机号；若用户拒绝，提示需绑定手机号才能继续使用 |
| 用户信息获取 | 获取微信头像和昵称作为默认值，用户可在个人中心修改 |
| 登录态维护 | 后端返回自定义登录态 token，存储在本地 storage；token 有效期7天，过期后自动重新登录 |
| 静默登录 | 已登录用户再次打开小程序时，自动检查 token 有效性，有效则静默登录，无效则重新走登录流程 |
| 登录态失效处理 | 接口返回401时，清除本地 token，跳转登录页面重新登录 |
| 新人标识 | 首次登录成功的用户标记为"新人"，用于新人礼包等营销活动判断 |

#### 3.1.3 业务规则

- 一个微信账号对应一个用户账号，不可重复注册
- 一个手机号只能绑定一个微信账号
- 手机号绑定后不可自行解绑，需联系客服处理
- 用户拒绝授权手机号时，仅可浏览商品，不可下单、加购等操作

#### 3.1.4 异常处理

- 微信登录接口超时：提示"网络异常，请重试"
- 手机号获取失败：提示"获取手机号失败，请重新授权"
- 服务器异常：提示"服务暂时不可用，请稍后再试"

### 3.2 首页

#### 3.2.1 功能描述

首页是用户进入小程序的第一个页面，承载商品展示、活动推广、内容引导等核心功能，通过模块化布局实现灵活运营。

#### 3.2.2 详细需求

| 需求项 | 描述 |
|-------|------|
| Banner 轮播 | 顶部轮播图，支持自动轮播（3秒间隔）和手动滑动；点击跳转至对应页面（商品详情/活动页/外部链接）；最多展示8张，最少1张 |
| 搜索入口 | 顶部搜索框，点击跳转搜索页面；搜索框内展示默认搜索词（后台可配置） |
| 推荐位 | Banner 下方展示2-4个推荐位图标入口，每个图标可配置跳转链接；用于活动入口、分类快捷入口等 |
| 活动入口 | 展示当前进行中的活动，以卡片形式展示，包含活动图片、名称、时间倒计时；点击进入活动详情页 |
| 热门商品 | 展示销量TOP商品，横向滑动或两列瀑布流展示；展示商品图片、名称、价格、销量标签 |
| 新品推荐 | 展示最近上架的商品，标识"新品"标签；展示商品图片、名称、价格 |
| 按月龄推荐 | 若用户已填写宝宝档案，根据宝宝月龄推荐适龄商品；展示月龄标签（如"6-8个月推荐"）和对应商品列表；若未填写宝宝档案，展示"完善宝宝档案，获取专属推荐"引导入口 |
| 公告栏 | 滚动公告信息，展示商城公告、发货通知等 |

#### 3.2.3 业务规则

- 首页所有模块的排序和显隐由后台"首页装修管理"控制
- 商品数据实时从接口获取，不使用本地缓存
- 下拉刷新更新首页数据
- 页面上拉加载更多商品

#### 3.2.4 交互说明

- 进入首页自动请求最新数据
- 下拉刷新：顶部出现加载动画，数据更新后自动收起
- Banner 点击：跳转至对应目标页面
- 商品点击：跳转至商品详情页

### 3.3 商品分类

#### 3.3.1 功能描述

提供商品的多级分类浏览功能，帮助用户快速定位目标商品。

#### 3.3.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 多级分类树 | 左侧展示一级分类列表，右侧展示当前选中一级分类下的二级分类及三级分类；支持最多三级分类 |
| 分类图标 | 每个一级分类配有图标，图标由后台上传配置 |
| 分类下商品数量 | 每个分类旁展示该分类下的在售商品数量 |
| 分类选中态 | 左侧当前选中一级分类高亮显示，右侧联动刷新 |
| 点击分类 | 点击最末级分类，跳转至该分类下的商品列表页 |
| 全部分类 | 默认选中第一个一级分类 |

#### 3.3.3 业务规则

- 分类数据由后台"商品分类管理"维护
- 仅展示有在售商品的分类
- 一级分类排序由后台配置
- 分类变更后前端实时更新

### 3.4 商品列表

#### 3.4.1 功能描述

展示某一分类或搜索条件下的商品列表，支持分页加载和多种筛选排序方式。

#### 3.4.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 分页加载 | 默认每页20条商品，上拉加载更多，加载完毕提示"没有更多了" |
| 筛选排序 | 支持综合排序（默认）、价格升序/降序、销量降序、新品优先 |
| 分类筛选 | 顶部展示当前分类路径面包屑；支持在当前分类下按二级/三级分类进一步筛选 |
| 商品卡片 | 展示商品主图、名称、价格、销量、"新品"/"热销"等标签 |
| 空状态 | 无商品时展示"暂无商品"空状态图 |
| 下拉刷新 | 下拉刷新重新加载第一页数据 |

#### 3.4.3 业务规则

- 仅展示已上架且有库存的商品
- 已售罄商品展示"已售罄"标识，不可加购
- 价格展示取 SKU 最低价，格式为"¥XX起"

### 3.5 商品搜索

#### 3.5.1 功能描述

提供关键词搜索功能，帮助用户快速找到目标商品。

#### 3.5.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 关键词搜索 | 输入关键词后点击搜索，跳转搜索结果列表页；支持按商品名称、品牌名搜索 |
| 搜索历史 | 展示最近20条搜索记录，按搜索时间倒序排列；支持单条删除和一键清空；搜索历史存储在本地 |
| 热门搜索 | 展示后台配置的热门搜索词，最多10个；点击直接搜索该关键词 |
| 搜索建议 | 输入过程中实时展示搜索建议词（防抖300ms），最多展示10条；点击建议词直接搜索 |
| 搜索结果 | 展示匹配的商品列表，复用商品列表页的筛选排序功能 |
| 空结果 | 无匹配结果时展示"未找到相关商品"提示，并推荐热门商品 |

#### 3.5.3 业务规则

- 搜索不区分大小写
- 搜索关键词前后空格自动去除
- 搜索历史最多保留20条，超出时删除最早的记录
- 相同关键词不重复记录

### 3.6 商品详情

#### 3.6.1 功能描述

展示单个商品的完整信息，是用户做出购买决策的核心页面。

#### 3.6.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 商品图片轮播 | 顶部展示商品图片轮播，支持手动滑动和指示器；最多10张图片；支持双指缩放查看大图 |
| SKU 选择 | 点击"选择规格"弹出 SKU 选择弹窗；展示所有规格项（如颜色、尺码）和可选值；选中规格后展示对应价格、库存；库存为0的规格置灰不可选；多规格时需全部选中才可加购 |
| 价格展示 | 展示当前选中 SKU 的售价；若有划线价则展示"原价 ¥XX"；会员价展示"会员价 ¥XX" |
| 库存展示 | 展示当前选中 SKU 的可售库存数量；库存低于10件时展示"仅剩X件"提示 |
| 详情富文本 | 商品详情以富文本形式展示，支持图片、文字、表格等；图片支持点击放大查看 |
| 规格参数 | 以表格形式展示商品规格参数（如品牌、产地、材质、适用年龄等） |
| 服务承诺 | 展示服务标签，如"正品保证"、"7天无理由退换"、"极速发货"等；点击标签展示详细说明 |
| 评价 | 展示商品评价摘要（好评率、总评价数）；展示最近3条评价（用户头像、昵称、评分、评价内容、评价图片、评价时间）；点击"查看全部评价"跳转评价列表页 |
| 加购/立即购买 | 底部固定操作栏：加入购物车、立即购买；点击弹出 SKU 选择弹窗（若未选规格） |
| 收藏 | 支持收藏/取消收藏商品；收藏状态实时同步 |
| 分享 | 支持分享商品给微信好友/群 |
| 客服 | 底部操作栏提供客服入口 |

#### 3.6.3 业务规则

- 商品下架后访问详情页提示"该商品已下架"
- 商品删除后访问详情页提示"该商品不存在"
- 价格以选中 SKU 为准，未选 SKU 时展示价格区间
- 评价仅展示审核通过的评价

### 3.7 购物车

#### 3.7.1 功能描述

管理用户已选商品，支持数量修改、规格修改、选择结算等操作。

#### 3.7.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 商品列表 | 展示购物车中的商品列表，每项展示：商品图片、名称、已选规格、单价、数量、小计金额 |
| 数量修改 | 支持点击"+"-"修改数量，数量不得低于1，不得超过库存上限 |
| 规格修改 | 点击规格文字弹出 SKU 选择弹窗，切换规格后更新价格和库存 |
| 全选/单选 | 顶部全选复选框，每个商品前有单选复选框；全选与单选联动 |
| 合计金额 | 底部展示已选商品总金额（不含运费）和已选商品数量 |
| 去结算 | 点击"去结算"跳转下单确认页，携带已选商品信息 |
| 删除商品 | 左滑或长按出现删除按钮，确认后删除 |
| 失效商品 | 已下架/已售罄商品置灰显示，标注"已失效"，不可勾选；支持一键清除失效商品 |
| 空购物车 | 购物车为空时展示空状态图和"去逛逛"按钮 |
| 角标 | 底部TabBar购物车图标展示商品数量角标（最大显示99+） |

#### 3.7.3 业务规则

- 购物车数据存储在服务端，换设备登录后数据同步
- 单个用户购物车商品种类上限100
- 单个商品数量上限99
- 商品价格以加购时的最新价格为准，进入购物车时刷新价格

### 3.8 下单确认

#### 3.8.1 功能描述

用户确认订单信息并提交订单的核心流程页面。

#### 3.8.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 收货地址选择 | 顶部展示默认收货地址，点击可切换其他地址；点击"新增地址"跳转地址编辑页；无地址时引导新增 |
| 商品清单 | 展示本次下单的商品列表（图片、名称、规格、单价、数量） |
| 优惠券选择 | 展示可用优惠券数量，点击弹出优惠券选择弹窗；展示每张优惠券的面额、使用条件、有效期；已选优惠券高亮；不可用优惠券置灰并说明原因 |
| 积分抵扣 | 展示当前可用积分和可抵扣金额；开关控制是否使用积分抵扣；积分抵扣规则：100积分=1元，单笔最多抵扣订单金额的30% |
| 运费计算 | 展示运费金额；满X元包邮（后台可配置包邮门槛）；未满包邮门槛展示"还差¥XX元包邮"提示 |
| 备注 | 支持输入订单备注，最多50字 |
| 价格明细 | 展示商品总额、运费、优惠券减免、积分抵扣、应付金额 |
| 提交订单 | 点击"提交订单"创建订单，跳转支付页面 |

#### 3.8.3 业务规则

- 提交订单时校验商品库存，库存不足时提示并标记对应商品
- 提交订单时校验商品是否仍在售，已下架商品提示并移除
- 提交订单时校验优惠券是否仍可用
- 订单创建后锁定库存15分钟，超时未支付自动取消并释放库存
- 同一用户同一商品限购时，超出限购数量提示

### 3.9 微信支付

#### 3.9.1 功能描述

集成微信支付能力，完成订单在线支付。

#### 3.9.2 详细需求

| 需求项 | 描述 |
|-------|------|

## 4. 管理后台功能需求

### 4.1 管理员登录

#### 4.1.1 功能描述

管理员通过账号密码登录管理后台。

#### 4.1.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 账号密码登录 | 输入管理员账号和密码，点击登录；后端验证账号密码正确性 |
| 验证码 | 登录页展示图形验证码，防止暴力破解；验证码输入错误3次后锁定账号15分钟 |
| 权限验证 | 登录成功后根据管理员角色加载对应权限菜单；无权限的菜单不展示 |

| 退出登录 | 点击退出清除 token，跳转登录页 |

#### 4.1.3 业务规则

- 密码长度8-20位，需包含字母和数字
- 连续输错密码5次锁定账号30分钟
- 同一账号同一时间仅允许一个终端登录
- 登录成功记录登录IP和时间

## 5. 非功能需求

| 安全领域 | 要求 |
|---------|------|

| 数据权限 | 管理后台严格权限控制；用户数据隔离；敏感操作二次验证 |
| 日志审计 | 关键操作记录审计日志；异常访问告警 |

| 商品数据 | 商品浏览量（PV/UV）、加购数、转化率、销量、销售额 | 实时 + T+1 |
| 用户数据 | 新增用户数、活跃用户数（DAU/MAU）、留存率、消费频次 | T+1 |
| 营销数据 | 优惠券领取量/使用量/使用率、活动参与人数、活动转化率、活动ROI | T+1 |
| 内容数据 | 内容浏览量、阅读完成率、关联商品点击率 | T+1 |
| 客服数据 | 咨询量、响应时间、满意度 | T+1 |
| 库存数据 | 库存周转率、缺货率、滞销商品占比 | T+1 |

#### 数据报表要求

- 日报：每日9:00前生成昨日数据报告
- 周报：每周一生成上周数据报告
- 月报：每月1日生成上月数据报告
- 所有报表支持导出 Excel/PDF
- 关键指标异常自动告警（环比变化超过30%）

## 7. V1.0 新增需求

### 7.1 裂变分享 【V1.0 必做】

#### 7.1.1 功能描述

在现有基础分享（商品卡片、活动卡片、海报）之上，新增裂变分享体系，通过邀请奖励机制驱动用户主动拉新，实现私域流量增长。

#### 7.1.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 邀请分享页 | 用户专属邀请页，展示邀请规则、邀请码/小程序码、邀请人数统计、已获奖励明细 |
| 分享海报升级 | 海报增加邀请人专属小程序码（带 inviterId 参数），扫码进入自动绑定邀请关系 |
| 邀请关系绑定 | 新用户通过分享链接/海报扫码进入小程序后，自动绑定与邀请人的关系，绑定后不可更改 |
| 裂变奖励规则 | 邀请人奖励：每成功邀请1位新用户注册并完成首单，邀请人获得积分奖励（后台可配置）；被邀请人奖励：通过邀请链接注册的新用户可获得新人专属优惠券（后台可配置） |
| 邀请统计 | 用户可在邀请页查看：累计邀请人数、已下单人数、已获奖励总额 |
| 防刷机制 | 同一设备/微信号仅可绑定一次邀请关系；后台可配置每日/每月邀请奖励上限；异常邀请行为（短时间大量邀请）触发风控审核 |

#### 7.1.3 业务规则

- 邀请关系仅一级，不做多级分销
- 被邀请人首单完成（确认收货）后，邀请人才可获得奖励
- 奖励以积分形式发放，积分到账后按积分规则使用和过期
- 本平台不做分销体系，裂变分享仅用于拉新激励

### 7.2 到店自提 【V1.0 必做】

#### 7.2.1 功能描述

在现有快递配送基础上，新增到店自提配送方式，用户下单时可选择快递配送或到店自提，选择自提后需选择自提点。

#### 7.2.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 配送方式选择 | 下单确认页新增配送方式切换：快递配送 / 到店自提；默认快递配送 |
| 自提点选择 | 选择"到店自提"后，展示自提点列表供用户选择；自提点展示：门店名称、地址、营业时间、联系电话、距离（如可获取） |
| 自提点详情 | 点击自提点可查看详细信息：门店图片、详细地址、营业时间、联系电话、地图导航 |
| 自提码 | 自提订单支付成功后，系统生成6位数字自提码；订单详情页展示自提码（大字号展示，支持刷新） |
| 自提提示 | 选择自提方式后，确认订单页展示提示："请于X天内到店自提（自提点营业时间内）"，超期未提货订单自动取消 |
| 订单详情展示 | 自提订单详情页展示：自提码、自提点信息（名称/地址/营业时间/电话）、自提状态 |
| 运费规则 | 自提订单免运费 |

#### 7.2.3 业务规则

- 自提订单不填写收货地址，改为选择自提点
- 自提码在订单支付成功后生成，6位随机数字
- 自提有效期：支付成功后7天（后台可配置），超期自动取消并退款
- 自提核销：门店管理员输入自提码或扫码核销
- 自提订单不支持修改配送方式（下单后不可切换）
- 自提订单售后：未核销可申请退款；已核销按正常售后流程

#### 7.3.1 功能描述

在现有客服入口基础上，增强客服配置能力，支持后台灵活配置客服方式和展示信息。

| 客服入口统一 | 小程序端客服入口统一调用后台配置的客服信息，入口位置：商品详情页底部操作栏、订单详情页、个人中心 |

#### 7.3.3 业务规则

- 未配置客服信息时，客服入口展示默认提示"暂无客服信息"
- 非工作时间展示配置的自动回复语

### 7.4 活动板块承载视频/文章内容流 【V1.0 必做】

#### 7.4.1 功能描述

将活动 Tab 从单纯营销活动升级为"活动/内容/视频/种草/优惠"的综合运营入口，支持文章和视频两种内容类型，内容可投放到活动板块。

#### 7.4.2 详细需求

| 需求项 | 描述 |
|-------|------|
| 活动 Tab 升级 | 活动 Tab 首页新增子 Tab：推荐、优惠、视频、文章、线下活动 |
| 内容类型扩展 | 内容管理支持文章（article）和视频（video）两种类型；视频内容支持视频URL和封面图 |
| 内容投放 | 后台内容编辑时，可设置投放位置（placement）：活动板块/育儿知识/首页推荐等，支持多选 |
| 视频播放 | 视频类型内容在活动板块和内容详情页支持内嵌播放 |

- 视频格式支持 MP4，视频大小不超过 200MB
- 内容关联商品最多10个
- 标签由后台统一管理，每篇内容最多5个标签
