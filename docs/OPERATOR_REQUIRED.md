# 运营/老板必须人工提供信息清单（上线阻塞）

以下信息严禁编造，必须由运营、法务或老板提供真实值，并通过安全渠道填写到私有环境或审核平台。

## 1. 小程序与支付密钥

1. 小程序 AppID
2. AppSecret
3. 商户号（MCH ID）
4. APIv3 Key
5. 商户私钥（`apiclient_key.pem`）
6. 微信支付平台证书（`wechatpay_platform.pem`）

## 2. 备案与主体资质

1. ICP/小程序备案号
2. 营业执照统一社会信用代码
3. 食品经营许可证或仅销售预包装食品备案凭证
4. 保健食品批准文号/备案号
5. 奶粉产品配方注册号或合规证明

## 3. 客服与售后运营信息

1. 客服电话最终值
2. 客服微信最终值
3. 退货地址最终值
4. 售后规则（退款/退货/换货口径）
5. 隐私政策/用户协议最终确认人（姓名 + 职责）

## 4. 填写要求

1. 真实敏感值仅写入本地 `.env.production`、服务器证书目录或微信/商户后台。
2. 不得提交 `.env.production`、真实密钥、证书私钥到 Git 仓库。
3. 文档中不得出现伪造证照编号、备案号、批准文号。

## 5. 新增运维必做项（补偿与证书）

1. 微信平台证书轮换时，需更新 `WECHAT_PLATFORM_CERT_PATH` + `WECHAT_PLATFORM_CERT_SERIAL_NO`，或配置 `WECHAT_PLATFORM_CERT_MAP`（多序列号映射）。
2. 需安排后台运营定期巡检“支付补偿任务”（已取消订单收到支付回调）并闭环处理。
3. 正式发版前必须真机验证：支付成功、退款成功、已取消订单误支付补偿流程。
